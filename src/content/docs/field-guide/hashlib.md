## Что это

`hashlib` вычисляет хеш — короткий детерминированный отпечаток
последовательности байтов. Для одинаковых байтов SHA-256 возвращает одинаковый
результат; после изменения хотя бы одного байта хеш обычно меняется.

Хеш сравнивает содержимое, но не объясняет его и не восстанавливает исходный
файл. Совпадение SHA-256 служит сильной проверкой равенства байтов, а не
доказательством авторства или подлинности источника.

## Когда использовать

Используйте SHA-256 для поиска файлов с одинаковым содержимым, обнаружения
изменений и проверки скачанного архива по опубликованной контрольной сумме.
Сначала отделим текст от байтов и проверим алгоритм на маленьком значении.
Затем перенесём ту же операцию на файл.

## Этап 1. Хешируем известные байты

Функция хеширования принимает `bytes`, а не `str`. Литерал с префиксом `b`
задаёт ASCII-байты напрямую; для обычного текста границу кодирования нужно
обозначить явно через `.encode("utf-8")`.

```python
import hashlib

data = "case note".encode("utf-8")
# sha256 создаёт объект хеша, а hexdigest возвращает 64 шестнадцатеричных знака.
digest = hashlib.sha256(data).hexdigest()

print(len(data))
print(digest)
```

Точный вывод:

```text
9
627a6c2140448938132bb2441d164fb27a07312738406eed9a76bd31b61a0f8f
```

`data` — последовательность из 9 байтов, а `digest` — строка из 64 символов,
удобная для сохранения и сравнения. Кодировка входит в определение входных
байтов: один и тот же текст, закодированный по-разному, может получить другой
хеш. Мы проверили границу `str → bytes` и получили ожидаемый отпечаток. Для
проверки реального материала теперь нужно взять байты из файла.

## Этап 2. Вычисляем отпечаток файла

`Path.read_bytes` читает содержимое без декодирования: переводы строк,
кодировка и служебные байты остаются частью входа SHA-256. Чтение не изменяет
файл; наблюдаемый результат программы — напечатанная контрольная сумма.

```python
from pathlib import Path

path = Path("note.txt")
# read_bytes сохраняет точные байты файла без преобразования в текст.
file_bytes = path.read_bytes()
file_digest = hashlib.sha256(file_bytes).hexdigest()

print(path.name)
print(len(file_bytes))
print(file_digest)
```

Если `note.txt` содержит ровно девять UTF-8-байтов `case note` без перевода
строки в конце, точный вывод будет таким:

```text
note.txt
9
627a6c2140448938132bb2441d164fb27a07312738406eed9a76bd31b61a0f8f
```

Совпадение с первым этапом подтверждает, что файл содержит те же девять байтов.
Если редактор добавит перевод строки, размер и хеш изменятся. Мы получили
отпечаток файла и проверили число входных байтов; для нескольких файлов
следующий шаг программы может сгруппировать пути по значению `file_digest`.

## Большие файлы

`read_bytes` помещает весь файл в память. Для большого архива обновляйте один
объект хеша блоками; значение `1024 * 1024` задаёт блок размером 1 МиБ.

```python
hasher = hashlib.sha256()

with Path("archive.zip").open("rb") as source:
    while chunk := source.read(1024 * 1024):
        # update добавляет очередной блок к тому же вычислению SHA-256.
        hasher.update(chunk)

print(hasher.hexdigest())
```

Режим `"rb"` сохраняет файловую границу в байтах. Цикл имеет побочный эффект
только для позиции чтения открытого файла и состояния `hasher`; содержимое
архива не меняется. Итоговый хеш совпадёт с хешем `read_bytes()` для того же
файла, но объём памяти останется ограничен размером одного блока.

## Типичные ловушки

- Хеш описывает байты, а не смысл содержимого: разные кодировки или окончания
  строк дают разные входы.
- `hexdigest()` возвращает строку, а `digest()` — сырые байты. Для текстового
  отчёта и контрольной суммы обычно нужен `hexdigest()`.
- SHA-256 не заменяет цифровую подпись. Злоумышленник, способный заменить файл
  и опубликованный рядом хеш, обойдёт такую проверку.
- Для проверки изменений сохраните предыдущий хеш отдельно, иначе сравнивать
  новое значение будет не с чем.

## Где встречается в расследованиях

В текущей первой арке SHA-256 не входит в обязательное решение. Используйте
эту страницу для самостоятельного расширения проверки входных файлов.
