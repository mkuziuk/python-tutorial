## Что это

Сортировка превращает набор результатов в понятный порядок: от самого сильного совпадения к слабому, от свежего файла к старому, от высокого риска к низкому.

## Когда использовать

Используйте `sorted()`, когда нужно получить новый отсортированный список. Параметр `key` говорит, по какому признаку сравнивать элементы.

Например, детектор совпадений уже вычислил оценки для двух фрагментов. Входные данные — список словарей формы `{"name": имя фрагмента, "score": оценка}`. Нужно построить новый список, где самая высокая оценка стоит первой, и при этом сохранить исходный список без изменений.

```python
scores = [{"name": "frag-01", "score": 0.82}, {"name": "frag-02", "score": 0.41}]
ranked = sorted(scores, key=lambda item: item["score"], reverse=True)
print(ranked)
print(scores)
```

Результат:

```text
[{'name': 'frag-01', 'score': 0.82}, {'name': 'frag-02', 'score': 0.41}]
[{'name': 'frag-01', 'score': 0.82}, {'name': 'frag-02', 'score': 0.41}]
```

`key` получает один словарь и возвращает его числовое поле `score`; именно эти числа сравнивает `sorted()`. Параметр `reverse=True` задаёт убывание, поэтому `0.82` оказывается перед `0.41`. Две одинаковые строки вывода подтверждают другую границу операции: `ranked` — новый список, а `scores` не изменился. Оценка здесь служит только мерой, заданной детектором, и сама по себе не является вероятностью нарушения.

## Мини-пример

Та же операция работает с кортежами. В списке ниже каждый элемент имеет форму `(имя файла, число совпадений)`. Индекс `1` в функции `key` выбирает число совпадений, а не имя файла.

```python
pairs = [("a.txt", 4), ("b.txt", 12), ("c.txt", 7)]
print(sorted(pairs, key=lambda pair: pair[1], reverse=True))
```

Результат:

```text
[('b.txt', 12), ('c.txt', 7), ('a.txt', 4)]
```

Вывод подтверждает убывающий порядок `12 → 7 → 4`. Мы получили ранжированный список пар; теперь первые элементы можно показать в отчёте как кандидатов для ручной проверки.

## Типичные ловушки

- Без `key` Python сортирует элементы по их естественному порядку, который не всегда совпадает с вашей логикой.
- `list.sort()` меняет список на месте, а `sorted()` возвращает новый список.
- Если сортируете числа, убедитесь, что они не случайно сохранены строками.
