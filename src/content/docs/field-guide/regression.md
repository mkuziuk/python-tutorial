## Задача

Регрессия предсказывает число: цену, время, температуру. Прогноз `ŷᵢ` сравнивается с истинным `yᵢ`, а [остаток](../numpy/#math-vocabulary) обычно определяют так:

```text
rᵢ = yᵢ − ŷᵢ
```

Положительный остаток означает недооценку, отрицательный — переоценку, если используется именно это направление. Подпишите определение рядом с графиком.

Индекс `i` просто нумерует объекты. `n` ниже означает их количество, `Σᵢ` — «сложить по всем объектам», вертикальные черты `|…|` — взять абсолютную величину, а `√` — квадратный корень.

Небольшой пример в тысячах рублей: истинные значения `[200, 300, 500]`, прогнозы `[220, 260, 450]`. Остатки равны `[-20, 40, 50]`: первый объект переоценён на 20, два других недооценены.

## MAE

```text
MAE = Σᵢ |yᵢ − ŷᵢ| / n
```

MAE — средняя абсолютная ошибка в единицах цели. Если цель California Housing хранится в сотнях тысяч долларов, преобразуйте результат в доллары и явно назовите множитель.

Для примера выше `MAE = (20 + 40 + 50) / 3 ≈ 36.7` тысячи рублей. Знаки исчезли, поэтому переоценка и недооценка не компенсировали друг друга.

## RMSE

```text
RMSE = √(Σᵢ(yᵢ − ŷᵢ)² / n)
```

Возведение в квадрат сильнее наказывает крупные ошибки. Если RMSE заметно выше MAE, у модели есть редкие большие промахи или тяжелый хвост ошибок.

В том же примере `RMSE = √((20² + 40² + 50²) / 3) ≈ 38.7` тысячи рублей. Квадраты временно меняют единицу, а корень возвращает исходную.

## R²

```text
R² = 1 − Σᵢ(yᵢ − ŷᵢ)² / Σᵢ(yᵢ − ȳ)²
```

`ȳ` читается как «игрек с чертой» и означает среднее истинных значений **в оцениваемой выборке**. Знаменатель — ошибка гипотетического ориентира, который для всех этих объектов предсказывает это среднее. `R² = 1` соответствует идеальным прогнозам; около нуля — этому ориентиру на той же выборке; значение может быть отрицательным. Например, `R² = 0.8` означает, что сумма квадратов ошибок модели составляет 20% от ошибки такого ориентира, а не что «80% прогнозов верны».

Это определение метрики, а не инструкция обучать baseline на test. Реальный `DummyRegressor` учит среднее только по train; при сдвиге между train и test его `R²` может отличаться от нуля и даже быть отрицательным.

```python
from sklearn.metrics import mean_absolute_error, r2_score, root_mean_squared_error

mae = mean_absolute_error(y_test, predictions)
rmse = root_mean_squared_error(y_test, predictions)
r2 = r2_score(y_test, predictions)
```

## Остатки

Средняя метрика не показывает, где модель ошибается. Проверьте:

- остаток против предсказания;
- абсолютную ошибку по диапазонам цели;
- карту/время/партию, если они относятся к deployment;
- ошибки у искусственной верхней или нижней границы target.

Идеальная «случайная полоса вокруг нуля» не является строгим тестом, но явный веер, изгиб или географический кластер указывает на систематическую структуру.

## Baseline и split

Сравните модели с `DummyRegressor`, предсказывающим среднее или медиану train. Оценивайте все варианты на одном split. Если задача обещает новый регион, случайный holdout соседних кварталов не проверяет этот claim.

## Типичные ловушки

- Называть `R² = 0.8` «80% правильных прогнозов».
- Путать единицы target и отчета.
- Считать только среднее и не видеть сегмент крупных ошибок.
- Удалять capped значения как «выбросы» без документации.
- Сравнивать MAE моделей на разных test-наборах.
