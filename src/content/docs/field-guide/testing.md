## Что такое тест

Модульный тест вызывает небольшой фрагмент программы с известными данными и сравнивает результат с ожидаемым. Тест не доказывает отсутствие всех ошибок, но быстро сообщает, сломали ли мы уже проверенное поведение.

```python
import unittest

from copy_paste_detector import make_ngrams


class NgramTests(unittest.TestCase):
    def test_builds_pairs(self):
        result = make_ngrams(["ночной", "сигнал", "архива"], size=2)
        self.assertEqual(
            result,
            [("ночной", "сигнал"), ("сигнал", "архива")],
        )
```

Имя метода начинается с `test_`, чтобы `unittest` нашел его автоматически. Один тест лучше посвящать одному наблюдаемому правилу.

## Как запускать

Из папки дела запустите весь комплект:

```bash
python -m unittest discover -s tests -v
```

Ключ `-v` показывает имя каждой проверки. Чтобы сосредоточиться на одном модуле или методе:

```bash
python -m unittest tests.test_copy_paste_detector -v
python -m unittest tests.test_copy_paste_detector.CopyPasteDetectorTests.test_make_ngrams_returns_word_tuples -v
```

## Как читать результат

- `OK` — все запущенные проверки прошли.
- `FAIL` — тест получил результат, но он отличается от ожидаемого. Ниже показаны оба значения и строка проверки.
- `ERROR` — код не смог завершить тест: например, импорт не нашел функцию или программа выбросила неожиданное исключение.

Читайте traceback снизу вверх: сначала найдите последнюю строку с типом ошибки, затем первую строку, которая относится к вашему файлу.

## Полезные проверки

```python
self.assertEqual(actual, expected)
self.assertAlmostEqual(score, 0.617)
self.assertIn("доступ", tags)
self.assertTrue(path.exists())

with self.assertRaises(ValueError):
    make_ngrams(["текст"], size=0)
```

Для файлов создавайте временную папку, чтобы тест не портил рабочие данные:

```python
from pathlib import Path
import tempfile

with tempfile.TemporaryDirectory() as tmp:
    output = Path(tmp) / "report.json"
    output.write_text("{}", encoding="utf-8")
    self.assertTrue(output.exists())
```

## Когда запускать

Пустой стартовый файл еще не содержит функций, которые импортируют итоговые тесты, поэтому полный комплект сначала выдаст `ERROR`. Запускайте его после добавления функций из главы, а между крупными шагами используйте промежуточные команды самой главы.

После исправления ошибки снова запустите все тесты: успешная отдельная проверка не гарантирует, что соседнее поведение осталось целым.

## Типичные ловушки

- Не меняйте ожидаемый результат только ради зеленого теста; сначала проверьте правило задачи.
- Тест не должен зависеть от порядка запуска других тестов.
- Не записывайте результаты теста поверх учебных данных.
- Проверяйте не внутренние строки реализации, а результат, который важен пользователю или следующей функции.
