## Что это

Регулярное выражение — это шаблон для поиска фрагментов текста. В первом
расследовании оно превращает одну строку в список русских слов без приклеенной
пунктуации.

## Когда использовать

Используйте `re`, если простого `.split()` мало: нужно найти слова, даты,
номера, теги или повторяющиеся текстовые формы. Сначала задайте границу данных:
какие символы считаются частью совпадения, а какие должны быть отброшены.

Ниже вход — обычная строка Python, уже декодированная в Unicode. Шаблон должен
найти только последовательности русских букв; пробелы и знаки препинания
останутся за пределами результата.

```python
import re

# Диапазон и буква ё задают допустимый алфавит одного слова.
word_re = re.compile(r"[а-яё]+", re.IGNORECASE)
words = word_re.findall("След есть. След рядом!")
print(words)
```

Результат:

```text
['След', 'есть', 'След', 'рядом']
```

`findall()` возвращает список строк в порядке появления. Вывод подтверждает,
что точка и восклицательный знак не вошли в слова, а `re.IGNORECASE` позволил
сохранить совпадение с заглавной буквой. Регистр самих строк при этом не
меняется.

Мы получили список слов и проверили границу допустимых символов. Если следующий
этап должен сравнивать слова без учёта регистра, его входом станет
нормализованный список:

```python
normalized = [word.casefold() for word in words]
print(normalized)
```

```text
['след', 'есть', 'след', 'рядом']
```

`casefold()` подготавливает Unicode-строки к сравнению без учёта регистра.
Теперь два появления слова `след` представлены одинаково и подходят, например,
для подсчёта частот.

## Типичные ловушки

- `.` в regex означает почти любой символ, а не точку. Для настоящей точки пишут `\.`
- `+` означает "один или больше", поэтому `[а-яё]+` склеивает соседние буквы в слово.
- `r"..."` не обязательно всегда, но помогает не запутаться с обратными слешами.
- `[а-яё]+` намеренно не включает латиницу, цифры и дефис: `e-mail` и `архив-17`
  распадутся на части или будут пропущены. Для другой задачи нужен другой
  шаблон и отдельные проверочные примеры.
- При чтении файла сначала явно декодируйте байты, например
  `Path(...).read_text(encoding="utf-8")`; модуль `re` работает со строкой, а не
  угадывает кодировку файла.
- Regex полезен, но не должен заменять нормальный парсер, если формат данных сложный.
