## Что это

`numpy.ndarray` — плотный многомерный массив одного основного типа. В машинном обучении строки обычно представляют объекты, а столбцы — признаки.

## Математический словарь Части II

Эти термины нужны для чтения кода и результатов; доказательства и высшая математика не требуются.

| Термин | Простое значение | Мини-пример |
| --- | --- | --- |
| **скаляр** | одно число | температура `18.5` |
| **вектор** | упорядоченный одномерный ряд чисел | один цветок `[5.1, 3.5, 1.4, 0.2]` |
| **координата / компонента** | одно место внутри вектора; в ML обычно один признак | первая координата `5.1` — длина чашелистика |
| **матрица** | прямоугольная таблица чисел; строки часто являются объектами | 150 цветков × 4 признака |
| **shape, или форма** | размеры массива по осям | `(150, 4)` означает 150 строк и 4 столбца |
| **расстояние** | число, показывающее, насколько различаются два вектора по выбранному правилу | расстояние 0 означает одинаковые координаты |
| **среднее** | сумма значений, делённая на их количество | среднее `[2, 4, 6]` равно 4 |
| **разброс** | насколько значения удалены друг от друга и от центра | `[4, 5, 6]` и `[0, 5, 10]` имеют среднее 5, но разный разброс |
| **вероятность / значение модели и порог** | вероятность лежит от 0 до 1; произвольное значение модели (`score`) может иметь другую шкалу; порог переводит выбранное число в класс | `0.72 >= 0.60`, значит ставим метку 1 |
| **остаток** | истинное число минус прогноз | `100 − 90 = +10`: модель недооценила |
| **интервал** | диапазон правдоподобных значений оценки при указанной процедуре | `[0.61, 0.69]` описывает неопределённость метрики, не гарантирует будущее |

Слово «разброс» обозначает идею, а измерять её можно по-разному: размахом `max − min`, стандартным отклонением или межквартильным диапазоном. Всегда называйте конкретную меру. Аналогично, «интервал» имеет смысл только вместе с методом расчёта и уровнем, например «95%-й бутстреп-интервал по источникам».

Вектор здесь не требует знаний физики — это удобная числовая запись одного объекта. Проверим конкретный вход: два цветка, описанные четырьмя измерениями. Ожидаемый результат — матрица формы `(2, 4)`, то есть две строки объектов и четыре столбца признаков:

```python
import numpy as np

X = np.array([
    [5.1, 3.5, 1.4, 0.2],
    [6.7, 3.1, 4.7, 1.5],
])

print(X.shape)  # (2, 4): два объекта, четыре признака
print(X.ndim)   # 2
print(X.dtype)  # обычно float64
```

```text
(2, 4)
2
float64
```

Вывод подтверждает, что `X` — двумерная таблица чисел: восемь значений распределены по двум объектам и четырём признакам. `float64` описывает способ хранения чисел, но ничего не говорит об их единицах или качестве измерения. Мы проверили общую форму; чтобы выбрать отдельные объекты и признаки, дальше нужны индексы и маски.

Всегда проговаривайте форму словами. Число `(1797, 64)` само по себе менее полезно, чем «1797 изображений, каждое представлено 64 пикселями».

Формы `(4,)` и `(1, 4)` различаются. Первая — одномерный вектор из четырёх чисел, вторая — двумерная матрица с одной строкой. Многие модели принимают только матрицу `X`, поэтому для одного объекта может понадобиться `vector.reshape(1, -1)`.

## Индексы и маски

Возьмём ту же структуру `(2, 4)` и спросим, какие объекты имеют первое измерение больше `6.0`. Вход — матрица цветков; ожидаемый результат — булева маска длины 2 и матрица только с подходящей строкой:

```python
import numpy as np

X = np.array([
    [5.1, 3.5, 1.4, 0.2],
    [6.7, 3.1, 4.7, 1.5],
])

first_object = X[0]
first_feature = X[:, 0]
large = X[:, 0] > 6.0
selected = X[large]

print(first_object.shape)
print(large)
print(selected.shape)
```

```text
(4,)
[False  True]
(1, 4)
```

`first_object` — вектор из четырёх признаков, `large` сопоставляет одно значение `True/False` каждой из двух строк, а `selected` сохраняет одну подходящую строку и все четыре столбца. Порог `6.0` выбран только для демонстрации и не доказывает полезность признака. Булева маска должна иметь длину выбранной оси: если `X` содержит 100 строк, маска объектов тоже должна содержать 100 значений. Мы отобрали строки; теперь проверим, как агрегировать значения по разным осям.

## Оси

Параметр `axis` говорит, какую ось «свернуть». Для матрицы из двух цветков сравним среднее по столбцам со средним по строкам; ожидаем соответственно вектор длины 4 и вектор длины 2:

```python
import numpy as np

X = np.array([
    [5.1, 3.5, 1.4, 0.2],
    [6.7, 3.1, 4.7, 1.5],
])

print(X.mean(axis=0))  # среднее каждого признака
print(X.mean(axis=1))  # одно среднее для каждого объекта
```

```text
[5.9  3.3  3.05 0.85]
[2.55 4.  ]
```

`axis=0` сворачивает две строки и оставляет четыре столбца, а `axis=1` сворачивает четыре столбца и оставляет две строки. Эти средние смешивают разные измерения цветка и поэтому служат только проверкой механики осей, а не содержательным анализом. Мы получили нужные длины; при переходе к изображениям та же проверка формы защищает от потери пикселей.

## Изображение и вектор признаков

В Digits одно изображение имеет форму `(8, 8)`, а классические модели получают плоский вектор длины 64. Проверим преобразование на массиве из чисел `0…63`: ожидаем сохранить 64 элемента и получить форму `(64,)`:

```python
import numpy as np

image = np.arange(64).reshape(8, 8)
features = image.reshape(-1)

assert image.shape == (8, 8)
assert features.shape == (64,)
print(image.size, features.shape, features[:4])
```

```text
64 (64,) [0 1 2 3]
```

`reshape()` сохранил 64 элемента и их порядок, изменив только форму представления. Искусственные числа здесь не являются настоящими яркостями Digits; пример проверяет лишь контракт формы. Для модели можно использовать `features`, а для показа изображения потребуется вернуть `features.reshape(8, 8)`.

## Векторные вычисления

Вместо цикла по каждому объекту NumPy выполняет операцию сразу над массивом. Для трёх истинных значений и трёх прогнозов вычислим абсолютные ошибки и RMSE; ожидаем один массив формы `(3,)` и одну итоговую величину:

```python
import numpy as np

y_true = np.array([3.0, 5.0, 8.0])
y_pred = np.array([2.0, 5.0, 10.0])

assert y_true.shape == y_pred.shape
absolute_errors = np.abs(y_true - y_pred)
rmse = np.sqrt(np.mean((y_true - y_pred) ** 2))

print(absolute_errors)
print(f"{rmse:.3f}")
```

```text
[1. 0. 2.]
1.291
```

Три позиции соответствуют трём объектам: второй прогноз точен, а третий ошибается на 2 единицы. RMSE равен `1.291` в единицах цели и сильнее реагирует на крупную ошибку из-за возведения в квадрат. Три учебных наблюдения не позволяют судить о качестве модели; сначала мы проверили правильность вычисления, а затем ту же операцию можно применять к отложенной выборке.

Векторная запись короче и обычно быстрее цикла, но не отменяет проверки формы. NumPy может применить broadcasting и получить математически допустимый, но смыслово неверный результат.

**Broadcasting** — правило, по которому NumPy временно повторяет меньший массив вдоль совместимой оси. Например, `X - X.mean(axis=0)` вычитает один вектор средних из каждой строки `X`. Это полезно; но если ожидалась поэлементная пара объектов, формы `(100, 1)` и `(100,)` могут неожиданно дать матрицу `(100, 100)`. Перед вычислением ошибки полезно проверить `y_true.shape == y_pred.shape`.

## Типичные ловушки

- Перепутать `(n,)` и `(n, 1)`; это разные формы.
- Случайно изменить исходный массив через view.
- Смешать индексы обучающей и тестовой выборок после отдельной сортировки `X` и `y`.
- Усреднить по неверной оси.
- Передать в изображение плоский вектор без `reshape(8, 8)`.
