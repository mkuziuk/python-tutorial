## Что это

`numpy.ndarray` — плотный многомерный массив одного основного типа. В машинном обучении строки обычно представляют объекты, а столбцы — признаки.

## Математический словарь Части II

Эти термины нужны для чтения кода и результатов; доказательства и высшая математика не требуются.

| Термин | Простое значение | Мини-пример |
| --- | --- | --- |
| **скаляр** | одно число | температура `18.5` |
| **вектор** | упорядоченный одномерный ряд чисел | один цветок `[5.1, 3.5, 1.4, 0.2]` |
| **координата / компонента** | одно место внутри вектора; в ML обычно один признак | первая координата `5.1` — длина чашелистика |
| **матрица** | прямоугольная таблица чисел; строки часто являются объектами | 150 цветков × 4 признака |
| **shape, или форма** | размеры массива по осям | `(150, 4)` означает 150 строк и 4 столбца |
| **расстояние** | число, показывающее, насколько различаются два вектора по выбранному правилу | расстояние 0 означает одинаковые координаты |
| **среднее** | сумма значений, делённая на их количество | среднее `[2, 4, 6]` равно 4 |
| **разброс** | насколько значения удалены друг от друга и от центра | `[4, 5, 6]` и `[0, 5, 10]` имеют среднее 5, но разный разброс |
| **вероятность / score и порог** | вероятность лежит от 0 до 1; score — численная уверенность модели и может иметь другую шкалу; порог переводит выбранное число в класс | `0.72 >= 0.60`, значит ставим метку 1 |
| **остаток** | истинное число минус прогноз | `100 − 90 = +10`: модель недооценила |
| **интервал** | диапазон правдоподобных значений оценки при указанной процедуре | `[0.61, 0.69]` описывает неопределённость метрики, не гарантирует будущее |

Слово «разброс» обозначает идею, а измерять её можно по-разному: размахом `max − min`, стандартным отклонением или межквартильным диапазоном. Всегда называйте конкретную меру. Аналогично, «интервал» имеет смысл только вместе с методом расчёта и уровнем, например «95% source-bootstrap interval».

Вектор здесь не требует знаний физики — это удобная числовая запись одного объекта. Несколько векторов можно сложить строками в матрицу:

```python
import numpy as np

X = np.array([
    [5.1, 3.5, 1.4, 0.2],
    [6.7, 3.1, 4.7, 1.5],
])

print(X.shape)  # (2, 4): два объекта, четыре признака
print(X.ndim)   # 2
print(X.dtype)  # обычно float64
```

Всегда проговаривайте форму словами. Число `(1797, 64)` само по себе менее полезно, чем «1797 изображений, каждое представлено 64 пикселями».

Формы `(4,)` и `(1, 4)` различаются. Первая — одномерный вектор из четырёх чисел, вторая — двумерная матрица с одной строкой. Многие модели принимают только матрицу `X`, поэтому для одного объекта может понадобиться `vector.reshape(1, -1)`.

## Индексы и маски

```python
first_object = X[0]
first_feature = X[:, 0]
large = X[:, 0] > 6.0
selected = X[large]
```

Булева маска должна иметь длину выбранной оси. Если `X` содержит 100 строк, маска объектов тоже должна содержать 100 значений `True/False`.

## Оси

Параметр `axis` говорит, какую ось «свернуть»:

```python
X.mean(axis=0)  # [5.9, 3.3, 3.05, 0.85]: среднее каждого признака
X.mean(axis=1)  # [2.55, 4.0]: одно среднее для каждого объекта
```

`axis=0` означает «свернуть строки и оставить столбцы», а `axis=1` — «свернуть столбцы и оставить строки». Удобная проверка: результат `axis=0` имеет длину числа признаков, а `axis=1` — числа объектов.

## Изображение и вектор признаков

В Digits одно изображение имеет форму `(8, 8)`, а классические модели получают плоский вектор длины 64:

```python
image = images[0]
features = image.reshape(-1)

assert image.shape == (8, 8)
assert features.shape == (64,)
```

`reshape()` меняет представление, а не порядок значений. Не вызывайте его вслепую: проверьте, что число элементов сохраняется.

## Векторные вычисления

Вместо цикла по каждому объекту NumPy выполняет операцию сразу над массивом:

```python
absolute_errors = np.abs(y_true - y_pred)
rmse = np.sqrt(np.mean((y_true - y_pred) ** 2))
```

Это короче и обычно быстрее, но не отменяет проверки формы. NumPy может применить broadcasting и получить математически допустимый, но смыслово неверный результат.

**Broadcasting** — правило, по которому NumPy временно повторяет меньший массив вдоль совместимой оси. Например, `X - X.mean(axis=0)` вычитает один вектор средних из каждой строки `X`. Это полезно; но если ожидалась поэлементная пара объектов, формы `(100, 1)` и `(100,)` могут неожиданно дать матрицу `(100, 100)`. Перед вычислением ошибки полезно проверить `y_true.shape == y_pred.shape`.

## Типичные ловушки

- Перепутать `(n,)` и `(n, 1)`; это разные формы.
- Случайно изменить исходный массив через view.
- Смешать индексы train/test после отдельной сортировки `X` и `y`.
- Усреднить по неверной оси.
- Передать в изображение плоский вектор без `reshape(8, 8)`.
