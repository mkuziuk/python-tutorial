## Задача: проверить состояние из внешних данных

Пусть JSON передаёт строку `support` или `conflict`. Обычная строка допускает
незаметную опечатку: `suport` не равно `support`, но программа узнает об ошибке
только тогда, когда не сработает нужная ветка. Разделим обработку на два этапа:

1. преобразуем внешнюю строку в значение из конечного набора;
2. сопоставим проверенное значение с правилом начисления баллов.

Для первого этапа подходит `StrEnum` — перечисление, значения которого ведут
себя как строки.

## Объявляем допустимые значения

Класс перечисления связывает понятные имена `SUPPORT` и `CONFLICT` со строками,
которые встречаются во входных данных:

```python
from enum import StrEnum


class Stance(StrEnum):
    SUPPORT = "support"
    CONFLICT = "conflict"
```

Получилась структура «имя участника перечисления → строковое значение».
Например, `Stance.SUPPORT.value` возвращает строку `"support"`. Конструктор
`Stance(...)` использует эту структуру как проверяемый словарь допустимых
значений.

## Проверяем входную строку

Смоделируем значение, уже прочитанное из JSON, и преобразуем его в `Stance`:

```python
raw_stance = "support"
stance = Stance(raw_stance)

print(stance is Stance.SUPPORT)
print(stance.value)
```

Результат:

```text
True
support
```

Первая строка подтверждает, что конструктор вернул именно участника
`Stance.SUPPORT`, а вторая показывает значение, которое можно записать обратно
в JSON. Неизвестная строка, например `Stance("maybe")`, вызывает `ValueError`.
Проверка не исправляет регистр и пробелы: решение о том, допустимо ли превращать
`" Support "` в `"support"`, нужно принять до создания `Stance`.

Мы получили проверенное значение `stance` и убедились, что оно принадлежит
конечному набору. Чтобы превратить состояние в баллы, теперь можно сопоставить
его с одним из двух участников перечисления.

## Выбираем действие с помощью `match`

`match` последовательно сравнивает значение с шаблонами и выполняет первую
подходящую ветку. Сначала объявим оба счётчика, чтобы пример запускался
самостоятельно:

```python
support_points = 0
conflict_points = 0

match stance:
    case Stance.SUPPORT:
        support_points += 3
    case Stance.CONFLICT:
        conflict_points += 3

print(support_points, conflict_points)
```

Результат:

```text
3 0
```

Входное значение `support` увеличило только `support_points`. Этот результат
зависит от правила примера: каждое состояние добавляет ровно три балла.

Мы получили пару счётчиков `(3, 0)` и проверили действие отдельного состояния.
Для итогового статуса нужно учитывать уже форму пары и отношения между двумя
числами, поэтому следующий `match` работает сразу с кортежем.

## Сопоставляем пару значений

В шаблоне имена `support` и `conflict` захватывают элементы пары. Дополнительное
условие после `if` называется сторожевым условием (guard): ветка выполняется,
только если оно истинно.

```python
support_points = 12
conflict_points = 6

match support_points, conflict_points:
    case 0, 0:
        status = "no_evidence"
    case support, conflict if support >= 10 and support >= conflict + 5:
        status = "supported"
    case support, conflict if conflict > support:
        status = "not_supported"
    case _:
        status = "unresolved"

print(status)
```

Результат:

```text
supported
```

Пара `(12, 6)` проходит два порога: поддерживающих баллов не меньше десяти, а
разница составляет не меньше пяти. Это пороги учебного правила, а не
универсальная оценка надёжности. `_` охватывает все пары, которые не подошли
раньше.

## Порядок веток и типичные ловушки

- Ставьте частные и строгие правила раньше общих: `match` выбирает только
  первую подходящую ветку.
- Оставляйте `case _` последним, иначе следующие варианты будут недостижимы.
- Сравнивайте значение со строкой `"support"`, а не с именем участника
  `"SUPPORT"`.
- Помните, что имя в шаблоне захватывает значение, а не сравнивает его с
  одноимённой переменной снаружи.
- Для простого сравнения одного числа цепочка `if` часто яснее. `match` полезен,
  когда важна форма данных или конечный набор вариантов.

## Где встречается в расследованиях

В текущей первой арке `StrEnum` и `match` не входят в обязательное решение.
Страница остаётся справкой для самостоятельных проектов.
