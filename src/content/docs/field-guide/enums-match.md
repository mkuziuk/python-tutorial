## Зачем ограничивать значения

Обычная строка допускает незаметные опечатки: `"suport"` не равно `"support"`. `StrEnum` объявляет конечный словарь допустимых строк и даёт каждому значению понятное имя:

```python
from enum import StrEnum


class Stance(StrEnum):
    SUPPORT = "support"
    CONFLICT = "conflict"
```

`Stance.SUPPORT` можно сравнивать со строкой `"support"`, загруженной из JSON. Для записи значения перечисления в JSON используйте `.value`.

## Создание и сохранение enum

Конструктор проверяет входную строку:

```python
stance = Stance("support")
print(stance is Stance.SUPPORT)  # True
print(stance.value)              # support
```

Неизвестное значение, например `Stance("maybe")`, вызывает `ValueError`. Это полезно: поврежденные данные останавливаются при загрузке, а не превращаются в неверный вердикт позже. При подготовке обычного словаря для JSON используйте `.value`, чтобы намерение было явным.

## `match` и варианты

`match` сравнивает значение с шаблонами и выполняет первую подходящую ветку:

```python
match stance:
    case Stance.SUPPORT:
        support_points += 3
    case Stance.CONFLICT:
        conflict_points += 3
```

Для нескольких значений шаблон повторяет их форму:

```python
match support_points, conflict_points:
    case 0, 0:
        status = "no_evidence"
    case support, conflict if support >= 10 and support >= conflict + 5:
        status = "supported"
    case support, conflict if conflict > support:
        status = "not_supported"
    case _:
        status = "unresolved"
```

`support` и `conflict` получают значения из пары. Часть после `if` называется guard: ветка подходит, только если дополнительное условие истинно. `_` означает любой оставшийся случай.

## Порядок веток

Проверяется первая подходящая ветка, поэтому частные и строгие правила ставьте раньше общих. Ветка `case _` должна быть последней: после нее остальные варианты недостижимы.

Для простого сравнения одного числа цепочка `if` часто яснее. `match` полезен, когда важна форма данных или есть небольшой конечный набор вариантов.

## Типичные ловушки

- Не сравнивайте enum с названием участника `"SUPPORT"`: его строковое значение — `"support"`.
- Конструктор enum чувствителен к регистру и пробелам; нормализуйте внешние данные осознанно.
- Переменные в шаблоне не сравнивают с одноименными переменными снаружи — они захватывают значения.
- Слишком широкий ранний `case` делает следующие ветки недостижимыми.

## Где встречается в расследованиях

В текущей первой арке `StrEnum` и `match` не входят в обязательное решение. Страница остаётся справкой для самостоятельных проектов.
