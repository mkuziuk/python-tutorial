## Что это

Класс описывает свой тип данных: какие поля у объекта есть и какие действия он умеет выполнять.

## Когда использовать

Используйте класс, когда записи имеют одинаковые поля, а операции должны изменять или проверять их состояние. Например, `CaseFile` хранит название и улики и предоставляет метод `add_evidence()`.

```python
class CaseFile:
    def __init__(self, title):
        self.title = title
        self.evidence = []

    def add_evidence(self, item):
        self.evidence.append(item)
```

У каждого экземпляра свой список `evidence`: его форма — `list[str]`, а метод
`add_evidence()` изменяет этот список, но ничего не возвращает.

## Мини-пример

```python
case = CaseFile("Анонимное письмо")
case.add_evidence("anonymous.txt")
print(case.evidence)
```

```text
['anonymous.txt']
```

Вывод подтверждает, что вызов метода сохранил имя файла в состоянии именно
объекта `case`. Когда одних полей и простого конструктора достаточно, ту же
модель можно записать короче с помощью `dataclass`.

## Типичные ловушки

- Класс нужен не для красоты, а когда данные и поведение действительно связаны.
- Метод всегда получает объект первым параметром, обычно `self`.
- Не превращайте один маленький скрипт в десяток классов без пользы.
