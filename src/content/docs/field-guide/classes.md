## Что это

Класс описывает свой тип данных: какие поля у объекта есть и какие действия он умеет выполнять.

## Когда использовать

Используйте класс, когда словарь начинает притворяться предметом: дело имеет название, статус, улики и методы для добавления новых материалов.

```python
class CaseFile:
    def __init__(self, title: str) -> None:
        self.title = title
        self.evidence = []

    def add_evidence(self, item: str) -> None:
        self.evidence.append(item)
```

## Мини-пример

```python
case = CaseFile("Анонимное письмо")
case.add_evidence("anonymous.txt")
print(case.evidence)
```

## Типичные ловушки

- Класс нужен не для красоты, а когда данные и поведение действительно связаны.
- Метод всегда получает объект первым параметром, обычно `self`.
- Не превращайте один маленький скрипт в десяток классов без пользы.
