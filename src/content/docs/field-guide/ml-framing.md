## Сначала вопрос, потом алгоритм

До кода запишите:

1. **объект прогноза** — цветок, пассажир, район или изображение;
2. **момент прогноза** — какие сведения уже существуют;
3. **цель** `y` — класс или число;
4. **признаки** `X` — допустимые входы;
5. **условие новых данных** — случайная строка, новый источник, регион или партия;
6. **цену ошибок** и метрику;
7. **baseline**, который модель должна превзойти.

Классификация предсказывает конечную метку, регрессия — числовое значение. Это различие задается практическим вопросом, а не названием алгоритма.

## Что означают X и y

`X` — матрица допустимых признаков: строки — объекты, столбцы — измерения объекта. `y` — вектор правильных ответов, по одному на строку `X`. Заглавная буква подчёркивает, что `X` двумерна; это соглашение, а не специальный синтаксис Python. Определения вектора, матрицы, формы и координаты есть в [математическом словаре](../numpy/#math-vocabulary).

```python
import numpy as np

X = np.array([
    [5.1, 3.5],  # объект 0: два измерения
    [4.9, 3.0],
    [5.0, 3.4],
    [5.2, 3.5],
    [6.7, 3.1],
    [5.8, 2.7],
    [6.4, 3.2],
    [6.0, 2.9],
])
y = np.array([0, 0, 0, 0, 1, 1, 1, 1])

assert X.shape[0] == y.shape[0]  # каждому объекту соответствует ответ
```

Для классификации элементы `y` — классы вроде `0/1` или названия видов. Для регрессии — числа вроде цены. Идентификатор строки может быть нужен для группировки и аудита, но обычно не является содержательным признаком.

## Train, validation и test

- **train** используется для обучения параметров;
- **validation** или folds CV помогают выбрать модель, признаки, гиперпараметры и порог;
- **test/holdout** открывается после выбора и оценивает всю зафиксированную процедуру.

```python
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    X,
    y,
    test_size=0.25,
    random_state=42,
    stratify=y,
)
```

`stratify=y` сохраняет примерно одинаковые доли классов. Он не предотвращает пересечение связанных источников — для этого нужны группы.

## Baseline

Baseline отвечает: «что получится без изучения сложной структуры?»

```python
from sklearn.dummy import DummyClassifier

baseline = DummyClassifier(strategy="most_frequent")
baseline.fit(X_train, y_train)
baseline.score(X_test, y_test)
```

Для регрессии `DummyRegressor(strategy="median")` может всегда предсказывать медиану train. Если сложная модель не превосходит baseline, ее нельзя оправдать одной красивой визуализацией.

Конкретный ориентир: если в train 70% объектов относятся к классу 0, `most_frequent` даст около 70% accuracy на выборке с такой же долей. Модель с 72% почти не улучшает эту отправную точку, а для редкого класса может иметь recall 0. Поэтому baseline сравнивают по той же метрике, которая соответствует цене ошибок.

## Accuracy как доля

Для `n` объектов accuracy — доля правильных ответов:

```text
accuracy = число правильных прогнозов / n
```

Она понятна, но может скрыть редкий класс и разные цены false positive/false negative. Поэтому постановка задачи включает метрику, а не заканчивается выбором `train_test_split`.

## Типичные ловушки

- Оценивать на train и называть это качеством модели.
- Перебирать идеи по test, пока цифра не понравится.
- Использовать случайный split, когда deployment требует нового человека, региона или устройства.
- Выбрать признаки, появляющиеся после целевого события.
- Сравнивать модель без dummy baseline.
