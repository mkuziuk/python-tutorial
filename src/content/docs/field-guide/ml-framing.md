## Сначала вопрос, потом алгоритм

До кода запишите:

1. **объект прогноза** — цветок, пассажир, район или изображение;
2. **момент прогноза** — какие сведения уже существуют;
3. **цель** `y` — класс или число;
4. **признаки** `X` — допустимые входы;
5. **условие новых данных** — случайная строка, новый источник, регион или партия;
6. **цену ошибок** и метрику;
7. **базовую модель**, которую основная модель должна превзойти.

Классификация предсказывает конечную метку, регрессия — числовое значение. Это различие задаётся практическим вопросом, а не названием алгоритма.

## Что означают X и y

`X` — матрица допустимых признаков: строки — объекты, столбцы — измерения объекта. `y` — вектор правильных ответов, по одному на строку `X`. Заглавная буква подчёркивает, что `X` двумерна; это соглашение, а не специальный синтаксис Python. Определения вектора, матрицы, формы и координаты есть в [математическом словаре](../numpy/#math-vocabulary).

Проверим вход из восьми объектов с двумя измерениями и бинарной целью. Ожидаемый результат — формы `(8, 2)` для таблицы признаков и `(8,)` для ответов:

```python
import numpy as np

X = np.array([
    [5.1, 3.5],  # объект 0: два измерения
    [4.9, 3.0],
    [5.0, 3.4],
    [5.2, 3.5],
    [6.7, 3.1],
    [5.8, 2.7],
    [6.4, 3.2],
    [6.0, 2.9],
])
y = np.array([0, 0, 0, 0, 1, 1, 1, 1])

assert X.shape[0] == y.shape[0]  # каждому объекту соответствует ответ
print("X:", X.shape, "y:", y.shape)
```

Вывод `X: (8, 2) y: (8,)` буквально означает восемь объектов, два признака на объект и один ответ на каждую строку. Он подтверждает согласованность размеров, но не допустимость признаков и не качество меток. Для классификации элементы `y` — классы вроде `0/1` или названия видов; для регрессии — числа вроде цены. Идентификатор строки может быть нужен для группировки и аудита, но обычно не является содержательным признаком.

## Обучающая, проверочная и тестовая выборки

- **обучающая выборка (train)** используется для настройки параметров;
- **проверочная выборка (validation)** или фолды кросс-валидации помогают выбрать модель, признаки, гиперпараметры и порог;
- **тестовая выборка (test, holdout)** используется после выбора и оценивает всю зафиксированную процедуру.

Для демонстрационного массива выше отделим два объекта в test и сохраним доли классов. Ожидаемый результат — шесть обучающих и два тестовых объекта без пересечения:

```python
import numpy as np
from sklearn.model_selection import train_test_split

X = np.array([
    [5.1, 3.5], [4.9, 3.0], [5.0, 3.4], [5.2, 3.5],
    [6.7, 3.1], [5.8, 2.7], [6.4, 3.2], [6.0, 2.9],
])
y = np.array([0, 0, 0, 0, 1, 1, 1, 1])

X_train, X_test, y_train, y_test = train_test_split(
    X,
    y,
    test_size=0.25,
    random_state=42,
    stratify=y,
)
print(X_train.shape, X_test.shape)
print(np.bincount(y_train), np.bincount(y_test))
```

```text
(6, 2) (2, 2)
[3 3] [1 1]
```

Обе части содержат равные количества классов: по три объекта каждого класса в train и по одному в test. На восьми специально сбалансированных строках это лишь проверка механики. `stratify=y` сохраняет примерно одинаковые доли классов, но не предотвращает пересечение связанных источников — для этого нужны группы. Разбиение зафиксировано; теперь можно обучить простой ориентир, не выбирая решения по test.

## Базовая модель

Базовая модель отвечает на вопрос: «что получится без изучения сложной структуры?» До вызова оценки выберем и назовём метрику: **accuracy (доля правильных ответов)** — число правильных прогнозов, делённое на число объектов. Она уместна в этом сбалансированном учебном примере, где ошибки двух классов считаются одинаково дорогими.

Вход — шесть обучающих и два тестовых объекта из предыдущего примера. `DummyClassifier(strategy="most_frequent")` всегда выбирает самый частый класс train; ожидаемый результат — явное значение accuracy и сами прогнозы:

```python
import numpy as np
from sklearn.dummy import DummyClassifier
from sklearn.metrics import accuracy_score
from sklearn.model_selection import train_test_split

X = np.array([
    [5.1, 3.5], [4.9, 3.0], [5.0, 3.4], [5.2, 3.5],
    [6.7, 3.1], [5.8, 2.7], [6.4, 3.2], [6.0, 2.9],
])
y = np.array([0, 0, 0, 0, 1, 1, 1, 1])
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42, stratify=y
)

baseline = DummyClassifier(strategy="most_frequent")
baseline.fit(X_train, y_train)
baseline_predictions = baseline.predict(X_test)
baseline_accuracy = accuracy_score(y_test, baseline_predictions)

print("test classes:", np.bincount(y_test))
print("predicted classes:", np.bincount(baseline_predictions, minlength=2))
print(f"baseline accuracy: {baseline_accuracy:.2f}")
```

```text
test classes: [1 1]
predicted classes: [2 0]
baseline accuracy: 0.50
```

Baseline правильно классифицировал один из двух объектов, поэтому accuracy равна `1 / 2 = 0.50`. Он ни разу не предсказал класс 1: даже на сбалансированных данных одно число скрывает характер ошибки. Два тестовых объекта дают крайне неустойчивую оценку; пример объясняет расчёт, а не устанавливает рабочий порог качества. Кандидатную модель нужно сравнить на том же разбиении и той же заранее выбранной метрике, а при разной цене ошибок — дополнительно по recall, precision или стоимости ошибок.

Для регрессии `DummyRegressor(strategy="median")` может всегда предсказывать медиану обучающей выборки. Если сложная модель не превосходит базовую, её нельзя оправдать одной красивой визуализацией.

Конкретный ориентир: если в обучающей выборке 70% объектов относятся к классу 0, `most_frequent` даст около 70% правильных ответов на выборке с такой же долей. Модель с 72% почти не улучшает эту отправную точку, а для редкого класса может иметь recall 0. Поэтому метрику называют до оценки и сравнивают baseline с кандидатами по одному и тому же правилу.

## Доля правильных ответов (accuracy)

Для `n` объектов accuracy — доля правильных ответов:

```text
accuracy = число правильных прогнозов / n
```

Она понятна, но может скрыть редкий класс и разные цены false positive/false negative. Поэтому постановка задачи включает метрику, а не заканчивается выбором `train_test_split`.

## Типичные ловушки

- Оценивать на обучающей выборке и называть это качеством модели.
- Перебирать идеи по тестовой выборке, пока цифра не понравится.
- Использовать случайное разбиение, когда условия применения требуют нового человека, региона или устройства.
- Выбрать признаки, появляющиеся после целевого события.
- Сравнивать модель без базового варианта `DummyClassifier` или `DummyRegressor`.
