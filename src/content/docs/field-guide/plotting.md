## Зачем нужен график

График отвечает на конкретный вопрос: как распределён признак, разделяются ли классы, где растут ошибки, какие классы путаются? Если вопрос нельзя назвать, визуализация, скорее всего, украшает отчёт вместо проверки гипотезы.

Первый вопрос: где сосредоточены известные возраста и есть ли редкие крайние значения? Вход — 12 учебных возрастов и один пропуск; ожидаемый результат — histogram только по известным значениям и напечатанные границы, по которым можно проверить чтение рисунка:

```python
import matplotlib.pyplot as plt
import pandas as pd

df = pd.DataFrame({
    "age": [18, 19, 20, 21, 22, 22, 23, 24, 35, 36, 70, 71, None],
})
known_age = df["age"].dropna()

fig, ax = plt.subplots(figsize=(7, 4))
ax.hist(known_age, bins=[0, 20, 40, 60, 80])
ax.set(
    title="Возраст пассажиров с известным значением",
    xlabel="Возраст, лет",
    ylabel="Число пассажиров",
)
print(f"shown={known_age.size}, missing={df['age'].isna().sum()}")
age_bins = pd.cut(
    known_age,
    [0, 20, 40, 60, 80],
    right=False,
    include_lowest=True,
)
print(known_age.groupby(age_bins, observed=False).size())
plt.show()
```

```text
shown=12, missing=1
age
[0, 20)      2
[20, 40)     8
[40, 60)     0
[60, 80)     2
```

Самый высокий столбец — `20–40` лет: в нём 8 из 12 показанных пассажиров.
`right=False` задаёт те же полуинтервалы `[0, 20)`, `[20, 40)`, что использует
`hist`, поэтому возраст 20 попадает во второй столбец и в ту же строку
проверочной таблицы. Пустой интервал `40–60` отделяет основную группу от двух
значений `70–71`, поэтому на этом маленьком наборе виден разрыв, но нельзя
объявлять его свойством полной популяции. Один пропуск исключён вызовом
`dropna`, а границы столбцов заданы вручную; при других `bins` форма изменится.
Мы буквально прочитали распределение одного признака, а следующий выбор графика
зависит от нового вопроса — о категории, паре чисел или группах.

## Распределение и связь

- histogram показывает форму одного числового признака;
- bar chart сравнивает количества или явно подписанные доли категорий;
- scatter plot показывает связь двух числовых переменных;
- boxplot помогает увидеть медиану, разброс и выбросы по группам.

В histogram диапазон разбивается на интервалы (`bins`), а высота столбца показывает число значений внутри интервала. В scatter plot одна точка обычно соответствует одной строке: её координаты берутся из двух столбцов. Это полезная проверка смысла осей до интерпретации формы.

Одна таблица может отвечать на разные вопросы: `hist(age)` — «какие возраста встречаются?», `bar(pclass)` — «сколько строк в каждом классе?», `scatter(longitude, latitude)` — «где находятся наблюдения?». Выбирайте тип после вопроса, а не по внешнему виду.

Следующий вопрос: сосредоточены ли крупные ошибки прогноза в одной части карты? Вход — четыре объекта отложенной выборки, их координаты, истинные цены и прогнозы; ожидаемый результат — таблица абсолютных ошибок в долларах и карта, где цвет каждой точки соответствует той же строке:

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

map_frame = pd.DataFrame({
    "Longitude": [-122.4, -122.3, -118.3, -118.2],
    "Latitude": [37.8, 37.7, 34.1, 34.0],
    "actual_usd": [400_000, 420_000, 260_000, 255_000],
    "predicted_usd": [390_000, 300_000, 250_000, 150_000],
})
map_frame["absolute_error_usd"] = np.abs(
    map_frame["actual_usd"] - map_frame["predicted_usd"]
)

fig, ax = plt.subplots(figsize=(6, 5))
points = ax.scatter(
    map_frame["Longitude"],
    map_frame["Latitude"],
    c=map_frame["absolute_error_usd"],
    s=8,
    alpha=0.55,
)
fig.colorbar(points, ax=ax, label="Абсолютная ошибка, USD")
ax.set(xlabel="Долгота", ylabel="Широта", title="География ошибок")
print(map_frame[["Longitude", "Latitude", "absolute_error_usd"]])
plt.show()
```

```text
   Longitude  Latitude  absolute_error_usd
0     -122.4      37.8               10000
1     -122.3      37.7              120000
2     -118.3      34.1               10000
3     -118.2      34.0              105000
```

У двух пар координат есть по одной крупной ошибке — `120 000` и `105 000` USD, — поэтому этот пример не показывает концентрацию ошибок только в одном месте. Карта показывает пространственную ассоциацию, но не объясняет её причину: четыре искусственные строки слишком малы для географического вывода. В реальном анализе координаты и ошибки нужно брать по одним тестовым позициям; затем сравнить распределения ошибок по регионам и размеры групп, не подбирая гипотезу многократным просмотром holdout.

## Диагностика модели

Для классификации полезны confusion matrix и график метрики по порогу. Для регрессии — `y_true` против `y_pred` и остатки против предсказания. Для изображений — галерея ошибок с истинной и предсказанной меткой.

Показывайте не только удачные примеры. Если отбираете строки, опишите правило отбора: «первые 16 ошибок по индексу» воспроизводимее, чем ручной выбор.

## Честная подпись

У каждого содержательного графика должны быть:

- заголовок с вопросом или срезом;
- названия осей и единицы;
- легенда, если цвет/форма кодируют группу;
- размер выборки в тексте или подписи;
- пояснение, относится ли график к обучающей, проверочной или тестовой выборке.

## Типичные ловушки

- Обрезать ось так, что маленькая разница выглядит огромной.
- Наложить тысячи непрозрачных точек и принять плотное пятно за структуру.
- Сравнивать доли без размера групп.
- Многократно строить графики по тестовой выборке во время подбора и тем самым подглядывать в неё.
- Делать причинный вывод из визуальной ассоциации.
