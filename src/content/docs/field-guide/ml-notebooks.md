## Что это

Тетрадь Jupyter соединяет выполняемые ячейки Python, текст, таблицы и графики в одном файле `.ipynb`. Это удобно для расследования: рядом с кодом остаются вопрос, наблюдение и интерпретация. Но ядро (kernel) хранит состояние между запусками, поэтому тетрадь легко сделать невоспроизводимой.

## Главная модель состояния

Когда вы выполняете ячейку, имена попадают в память ядра. Проверим порог на трёх оценках модели: видимый код должен получить метки `[False, True, True]`. Порядок на странице и порядок выполнения могут различаться:

```python
# Ячейка 1
import numpy as np

probabilities = np.array([0.40, 0.70, 0.85])
threshold = 0.70

# Ячейка 2
labels = probabilities >= threshold
print(labels)
```

Вывод `[False  True  True]` подтверждает, что сравнение использовало порог `0.70`: равное порогу значение тоже получило метку `True`. Эти три искусственные оценки проверяют состояние, но не обосновывают выбор порога. Если сначала выполнить ячейку 2 после старого `threshold = 0.50`, результат не соответствует видимому коду. Номера `In [ ]` помогают заметить нарушение, но надёжная проверка одна: **Restart Kernel and Run All**. После чистого запуска можно интерпретировать результаты следующих ячеек.

## Локальный запуск

После распаковки ZIP откройте папку расследования в системном файловом приложении (Finder, Проводник или файловый менеджер Linux). Щёлкните по самой папке правой кнопкой и выберите **«Открыть в Терминале»**, **«Открыть в консоли»** или похожую команду. Так терминал сразу начнёт работу в нужной папке; вводить `cd` и угадывать путь не потребуется.

Задача этапа — создать изолированное окружение `.venv`, установить зафиксированные зависимости проекта и запустить Jupyter. Вход — распакованная папка с `requirements.txt`; ожидаемый результат — адрес Jupyter в терминале и список файлов проекта в браузере:

```bash
python3 -m venv .venv
source .venv/bin/activate        # macOS / Linux
python -m pip install -r requirements.txt
jupyter lab
```

В Windows PowerShell активация выглядит так:

```powershell
py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
jupyter lab
```

Команда `venv` создаёт папку `.venv`, активация направляет `python` и `pip` в неё, а установка читает версии из `requirements.txt`. Отдельно обновлять `pip` для учебного запуска не требуется. После `jupyter lab` терминал обычно печатает локальный адрес вида `http://localhost:8888/lab`; откройте его, если браузер не открылся сам.

Если `requirements.txt` не найден, терминал открыт не в корне расследования: вернитесь в файловое приложение и повторите правый щелчок по нужной папке. Относительный путь `data/titanic.csv` тогда одинаков локально и в Colab. Мы проверили среду и рабочую папку; следующий шаг — открыть `.ipynb` и выполнить его сверху вниз.

## Запуск в Colab

Ссылка из брифинга открывает на GitHub каноническую тетрадь для самостоятельной работы. Первая ячейка:

1. определяет среду;
2. загружает соответствующий ZIP;
3. проверяет SHA-256;
4. распаковывает ту же структуру папок;
5. выставляет рабочий каталог.

Выполните начальную ячейку один раз, затем выберите **Runtime → Run all**. Среда Colab временная: после сброса сеанса загруженные файлы исчезают, поэтому сохраняйте заполненный `.ipynb`, а не рассчитывайте на диск сессии.

## Хорошая учебная тетрадь

- одна ячейка делает один логический шаг;
- импорты и `RANDOM_STATE` объявлены в начале;
- данные не скачиваются из меняющегося источника во время анализа;
- график следует сразу за расчётом и имеет подписи;
- Markdown объясняет наблюдение, но не выдаёт догадку за факт;
- последняя ячейка содержит структурированное аудиторское мемо.

## Типичные ловушки

- Выполнять ячейки снизу вверх и забывать о старых переменных.
- Исправить функцию, но не перезапустить ячейки, которые ее вызывали.
- Записать абсолютный путь вроде `/Users/name/data.csv`.
- Установить пакет без фиксированной версии и получить другой результат через месяц.
- Очистить код, но оставить вывод от предыдущей версии.

Перед сдачей перезапустите ядро, выполните все ячейки сверху вниз и убедитесь, что нет ручных действий вне тетради.
