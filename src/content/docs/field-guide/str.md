## Что это

`str` хранит текст. В наших расследованиях строка часто является сырой уликой: письмо, лог, имя файла, строка отчёта.

## Когда использовать

Используйте `str`, когда нужно хранить или подготовить текст: привести к нижнему регистру, найти слова, проверить окончание файла, собрать сообщение для отчёта.

```python
text = "След оставлен ночью."
clean = text.lower()
print(clean)
```

```text
след оставлен ночью.
```

`clean` — новая строка; исходный `text` не меняется. Нижний регистр позволяет
сравнивать одинаково написанные слова без учёта регистра, но ещё не отделяет
слова от пунктуации. Для этого нужен следующий шаг подготовки.

## Мини-пример

Для конкретной фразы достаточно удалить известную точку и разбить строку по
пробелам:

```python
message = "Пауза была слишком длинной."
words = message.lower().replace(".", "").split()
print(words)
```

```text
['пауза', 'была', 'слишком', 'длинной']
```

Результат подтверждает форму данных: `words` — список четырёх строк. Это
узкое решение для этой фразы, а не общая токенизация: `replace(".", "")` не
удалит запятые, тире и другие знаки. Полученный список уже можно перебирать, а
если требуется только проверить имя файла, строку делить не нужно.

## Ещё примеры

Проверить формат строки можно без регулярных выражений:

```python
filename = "report_2026.txt"

if filename.startswith("report_") and filename.endswith(".txt"):
    print("похож на текстовый отчёт")
```

```text
похож на текстовый отчёт
```

Обе проверки истинны для `report_2026.txt`: имя имеет нужный префикс и
расширение. Такая проверка отвечает только на вопрос о форме имени; чтобы
показать человеку числовой результат анализа, соберём отдельное сообщение.

Для читаемого сообщения подставляйте значения через f-строку. Выражение внутри фигурных скобок вычисляется в момент создания новой строки:

```python
author = "Морозова"
score = 0.77
summary = f"Кандидат: {author}; сходство: {score:.0%}"
print(summary)
```

```text
Кандидат: Морозова; сходство: 77%
```

Спецификатор `.0%` превратил долю `0.77` в процент без знаков после запятой.
Он меняет только представление числа в `summary`, а не значение `score`.

## Типичные ловушки

- `split()` без аргумента разделяет по пробельным символам — пробелам, табуляции и переносам строки, — но не убирает пунктуацию.
- `lower()` не меняет исходную строку, а возвращает новую.
- `replace()` заменяет точное сочетание символов и тоже возвращает новую строку.
- Индексы начинаются с нуля.
