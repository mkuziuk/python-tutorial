## Что это

`set` хранит уникальные значения. Порядок не важен, зато пересечения и объединения работают коротко и быстро.

## Когда использовать

Берите множество, когда нужно узнать, что встречается хотя бы один раз, или сравнить два набора признаков.

```python
left = {"след", "пауза", "ночь"}
right = {"пауза", "запись", "ночь"}
print(sorted(left & right))
```

```text
['ночь', 'пауза']
```

Пересечение оставило два значения, присутствующие в обоих наборах. `sorted()`
нужно только для стабильного вывода; сами `left` и `right` остаются
множествами. Далее используем уникальность множества для одного списка.

## Мини-пример

```python
words = ["след", "пауза", "след"]
unique_words = set(words)
print(len(unique_words))
```

```text
2
```

Три входных слова превратились в два уникальных значения: повтор `"след"`
схлопнулся. Когда нужно не только посчитать, но и объяснить расхождение двух
наборов, применяют разность и объединение.

## Ещё примеры

Операции над множествами отвечают на разные вопросы:

```python
expected = {"manifest.json", "timeline.txt", "access.log"}
found = {"manifest.json", "access.log", "draft.txt"}

missing = expected - found       # ожидали, но не нашли
unexpected = found - expected    # нашли, но не ожидали
all_names = expected | found     # встретились хотя бы в одном наборе

print(sorted(missing))
print(sorted(unexpected))
print(sorted(all_names))
```

```text
['timeline.txt']
['draft.txt']
['access.log', 'draft.txt', 'manifest.json', 'timeline.txt']
```

Разности разделили пропущенное и лишнее имя, а объединение собрало четыре
уникальных имени. Сортировка делает только отчёт детерминированным. Для
проверки одного конкретного значения строить эти производные множества не
нужно — достаточно операции `in`.

Проверка принадлежности читается почти как обычная фраза:

```python
risky_extensions = {".exe", ".scr", ".js"}
if ".exe" in risky_extensions:
    print("нужна дополнительная проверка")
```

```text
нужна дополнительная проверка
```

Строка появилась, потому что `".exe"` входит в набор расширений. Такая
проверка подтверждает только принадлежность, а не безопасность самого файла.

## Типичные ловушки

- Множество не хранит порядок.
- Вывод множества может идти в разном порядке между запусками; сортируйте его только для отчёта: `sorted(values)`.
- Одинаковые элементы остаются в одном экземпляре.
- Как и ключи словаря, элементы множества должны быть хешируемыми.
