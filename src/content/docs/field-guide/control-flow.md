## Что это

Условия выбирают, какой код выполнить, а циклы повторяют один шаг для нескольких значений. В делах условия отбрасывают неподходящие данные, а циклы проходят по словам, письмам, файлам и уликам.

```python
score = 7

if score >= 7:
    verdict = "высокий риск"
elif score >= 3:
    verdict = "проверить вручную"
else:
    verdict = "низкий риск"
```

Python определяет границы ветки или цикла по отступу. Код с тем же отступом принадлежит одному блоку.

## Цикл `for`

`for` берет элементы последовательности по одному. Имя после `for` существует только как текущий элемент:

```python
files = ["note.txt", "report.txt"]

for filename in files:
    print(filename)
```

Если нужен порядковый номер, используйте `enumerate()`. Параметр `start=1` делает нумерацию удобной для отчёта:

```python
for position, filename in enumerate(files, start=1):
    print(position, filename)
```

`range(stop)` создаёт последовательность целых чисел от нуля до `stop`, не включая его. Это полезно, когда алгоритму нужны позиции, например начала n-грамм:

```python
words = ["ночной", "сигнал", "архива"]
for index in range(len(words) - 1):
    print(words[index], words[index + 1])
```

## Цикл `while`

`while` повторяется, пока условие истинно. В отличие от `for`, он подходит, когда заранее неизвестно число повторов:

```python
attempts = 3
while attempts > 0:
    print("Осталось попыток:", attempts)
    attempts -= 1
```

В деле 04 выражение `while chunk := file.read(chunk_size)` сначала читает очередной блок в `chunk`, а затем проверяет его. Пустые байты означают конец файла и останавливают цикл. Оператор `:=` называется выражением присваивания; используйте его только когда совместная запись действительно делает цикл яснее.

## Истинность значений

Пустые строки, списки, словари, множества, `None` и число `0` считаются ложными. Поэтому `if not words` означает «если список слов пуст»:

```python
if not words:
    raise ValueError("В тексте нет слов")
```

Операторы `and`, `or` и `not` соединяют и переворачивают условия. Сравнения можно объединять: `1 <= reliability <= 5` проверяет обе границы.

## Типичные ловушки

- После `if`, `elif`, `else`, `for` и `while` нужны двоеточие и блок с отступом.
- Изменение списка во время прохода по нему часто пропускает элементы; обычно создавайте новый результат.
- У `range()` правая граница не входит в результат.
- Убедитесь, что условие `while` когда-нибудь станет ложным, иначе цикл не закончится.
