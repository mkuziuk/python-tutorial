## Что это

Условия выбирают, какой код выполнить, а циклы повторяют один шаг для нескольких значений. В расследованиях условия отбрасывают неподходящие данные, а циклы проходят по словам, письмам, файлам и уликам.

```python
score = 7

if score >= 7:
    verdict = "высокий риск"
elif score >= 3:
    verdict = "проверить вручную"
else:
    verdict = "низкий риск"

print(verdict)
```

```text
высокий риск
```

При `score = 7` срабатывает первая подходящая ветка, поэтому `verdict` получает
строку `"высокий риск"`, а остальные ветки уже не проверяются. Когда одно
действие нужно применить к нескольким входам, вместо повторения условий
используют цикл.

Python определяет границы ветки или цикла по отступу. Код с тем же отступом принадлежит одному блоку.

## Цикл `for`

`for` берет элементы последовательности по одному. Имя после `for` существует только как текущий элемент:

```python
files = ["note.txt", "report.txt"]

for filename in files:
    print(filename)
```

```text
note.txt
report.txt
```

Две входные строки дали две итерации в исходном порядке. Если результат должен
показывать место каждого файла в отчёте, к элементу добавляют номер.

Если нужен порядковый номер, используйте `enumerate()`. Параметр `start=1` делает нумерацию удобной для отчёта:

```python
files = ["note.txt", "report.txt"]

for position, filename in enumerate(files, start=1):
    print(position, filename)
```

```text
1 note.txt
2 report.txt
```

`enumerate()` создаёт пары `номер, значение`, а `start=1` задаёт номера для
человека, не индексы списка. Для алгоритма, которому нужны именно индексы
соседних элементов, удобнее `range()`.

`range(stop)` создаёт последовательность целых чисел от нуля до `stop`, не включая его. Это полезно, когда алгоритму нужны позиции, например начала n-грамм:

```python
words = ["ночной", "сигнал", "архива"]
for index in range(len(words) - 1):
    print(words[index], words[index + 1])
```

```text
ночной сигнал
сигнал архива
```

Три слова образуют две соседние пары; правая граница `len(words) - 1` не
включается и не позволяет обратиться за концом списка. Здесь число повторов
известно заранее. Если оно определяется изменяющимся состоянием, нужен
`while`.

## Цикл `while`

`while` повторяется, пока условие истинно. В отличие от `for`, он подходит, когда заранее неизвестно число повторов:

```python
attempts = 3
while attempts > 0:
    print("Осталось попыток:", attempts)
    attempts -= 1
```

```text
Осталось попыток: 3
Осталось попыток: 2
Осталось попыток: 1
```

После каждой итерации `attempts` уменьшается, а при значении `0` условие
становится ложным. Этот результат подтверждает условие завершения; похожим
образом чтение файла завершается на пустом блоке.

В расследовании 04 выражение `while chunk := file.read(chunk_size)` сначала читает очередной блок в `chunk`, а затем проверяет его. Пустые байты означают конец файла и останавливают цикл. Оператор `:=` называется выражением присваивания; используйте его только когда совместная запись действительно делает цикл яснее.

## Истинность значений

Пустые строки, списки, словари, множества, `None` и число `0` считаются ложными. Поэтому `if not words` означает «если список слов пуст»:

```python
words = []

if not words:
    print("В тексте нет слов")
```

```text
В тексте нет слов
```

Пустой список сделал условие истинным. В рабочей функции вместо сообщения
можно выбросить `ValueError`, чтобы не продолжать анализ без входных слов.
Операторы `and`, `or` и `not` позволяют составить более точное условие.

Операторы `and`, `or` и `not` соединяют и переворачивают условия. Сравнения можно объединять: `1 <= reliability <= 5` проверяет обе границы.

## Типичные ловушки

- После `if`, `elif`, `else`, `for` и `while` нужны двоеточие и блок с отступом.
- Изменение списка во время прохода по нему часто пропускает элементы; обычно создавайте новый результат.
- У `range()` правая граница не входит в результат.
- Убедитесь, что условие `while` когда-нибудь станет ложным, иначе цикл не закончится.
