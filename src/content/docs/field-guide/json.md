## Что это

JSON — текстовый формат для обмена данными между программами. Модуль `json`
преобразует поддерживаемые контейнеры Python в JSON-текст и обратно:
`dict` становится объектом JSON, `list` — массивом, а строки, числа,
`True`, `False` и `None` — соответствующими простыми значениями JSON.

Граница здесь важна: JSON-текст имеет тип `str`, но файл хранит байты.
Параметр `encoding="utf-8"` задаёт, как `str` превращается в байты при записи
и восстанавливается при чтении.

## Когда использовать

Используйте JSON для manifest-файлов, настроек, небольших учебных наборов
данных и передачи состояния между расследованиями. Сначала сформируем словарь
с известной структурой, затем проверим полученный текст и только после этого
сохраним его в файл.

## Этап 1. Превращаем контейнер в текст

Входные данные — словарь `case_id → str` и `open → bool`. `json.dumps`
возвращает строку и не создаёт файл, поэтому результат можно проверить до
любого побочного эффекта.

```python
import json

payload = {"case_id": "case-05", "open": True}
# dumps возвращает JSON-текст; ensure_ascii=False сохраняет читаемые символы.
text = json.dumps(payload, ensure_ascii=False, indent=2)
print(text)
```

Точный вывод:

```text
{
  "case_id": "case-05",
  "open": true
}
```

Контейнер Python имел форму `имя поля → значение`, а `text` теперь имеет тип
`str`. Обратите внимание на `true`: это значение JSON, а не написание
логического значения `True` в Python. Мы получили читаемый текст и проверили
его структуру. Чтобы передать состояние другой программе, нужен файл, поэтому
следующий этап записывает эту строку на диск.

## Этап 2. Записываем и снова загружаем данные

`Path.write_text` создаёт или полностью перезаписывает `case.json` — это
побочный эффект. Затем `read_text` возвращает строку, а `json.loads`
восстанавливает из неё контейнер Python.

```python
from pathlib import Path

path = Path("case.json")
# UTF-8 задаёт одинаковую границу кодирования для записи и чтения.
written_chars = path.write_text(text, encoding="utf-8")
loaded = json.loads(path.read_text(encoding="utf-8"))

print(written_chars)
print(loaded)
print(type(loaded).__name__)
```

Для текста из первого этапа точный вывод:

```text
42
{'case_id': 'case-05', 'open': True}
dict
```

Число `42` — количество записанных символов, а не размер файла в байтах:
для не-ASCII-символов эти величины могут различаться. В данном примере
`loaded` снова имеет форму `имя поля → значение`, и поле `open` восстановлено
как `bool`. Мы проверили полный путь `dict → str → файл → str → dict`;
теперь контейнер можно передать следующему этапу программы.

## Типичные ловушки

- JSON не сохраняет произвольные объекты Python. Экземпляр класса, `Path`,
  `set` или `datetime` сначала нужно превратить в поддерживаемые значения.
- Ключи объектов JSON — строки. Числовой ключ словаря после кругового
  преобразования станет строкой.
- `json.dumps` и `json.loads` работают со строками; `json.dump` и `json.load`
  работают с уже открытыми файлами.
- Для русского текста обычно используют `ensure_ascii=False`, а файл явно
  читают и записывают в UTF-8.
- Запись через `write_text` заменяет прежнее содержимое файла. Если старые
  данные нужны, их следует сначала загрузить и объединить с новыми.
- JSON из внешнего источника может иметь правильный синтаксис, но неправильные
  поля и типы. `json.loads` не проверяет схему предметных данных.
