В Части II модель — один компонент проверяемой процедуры. Практическая задача этой страницы — увидеть, какой результат возвращает каждое семейство моделей и от каких предпосылок он зависит. Короткие примеры используют маленькие учебные массивы, чтобы результат можно было проверить глазами. В расследовании начинайте с базового варианта, собирайте подготовку данных в `Pipeline` и сравнивайте кандидатов на одинаковых разбиениях.

## k ближайших соседей (k-NN)

k-NN не строит явную формулу. Для нового объекта он находит `k` ближайших обучающих объектов и голосует их классами. Слова «вектор», «координата» и «расстояние» разобраны в [математическом словаре](../numpy/#math-vocabulary). Для двух векторов расстояние Евклида — длина разности:

```text
d(x, z) = √Σⱼ(xⱼ − zⱼ)²
```

Здесь `x` и `z` — два объекта с одинаковым набором признаков, индекс `j` перебирает признаки, а `Σ` означает «сложить». Для `x = [1, 2]` и `z = [4, 6]` разности равны `−3` и `−4`, поэтому расстояние равно `√(9 + 16) = 5`. Чем меньше число, тем ближе объекты в выбранном пространстве признаков.

Проверим это поведение на четырёх объектах с двумя признаками. `StandardScaler` вычислит масштаб только по `X_train`, а классификатор вернёт метку для нового объекта `[0.9, 1.0]`.

```python
from sklearn.neighbors import KNeighborsClassifier
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler

X_train = [[0.0, 0.0], [0.2, 0.1], [1.0, 1.0], [1.2, 0.9]]
y_train = [0, 0, 1, 1]

knn = make_pipeline(
    StandardScaler(),
    KNeighborsClassifier(n_neighbors=3),
)
knn.fit(X_train, y_train)

print(knn.predict([[0.9, 1.0]]))
```

```text
[1]
```

Массив результата имеет одну метку: большинство из трёх ближайших соседей нового объекта относится к классу `1`. Масштабирование важно: признак с большими числами иначе доминирует в расстоянии. Например, разница в возрасте на 10 лет численно меньше разницы в стоимости билета на 100 единиц и почти исчезнет без общего масштаба. Малое `k` чувствительно к шуму, слишком большое сглаживает полезную локальную структуру.

Мы получили метку на основе локального соседства и проверили роль масштаба. Если классы лучше разделяет общая линейная граница, следующий кандидат — логистическая регрессия.

## Логистическая регрессия

Несмотря на название, это классификатор. Он складывает признаки с весами, преобразует результат в оценку класса от 0 до 1 и применяет порог.

До преобразования модель вычисляет линейное значение:

```text
score = b + w₁x₁ + w₂x₂ + …
```

`x₁, x₂` — значения признаков, `w₁, w₂` — коэффициенты, выученные по обучающей выборке, `b` — свободный член. Метод `decision_function` возвращает это неограниченное линейное значение, а `predict_proba` преобразует его в две модельные оценки от 0 до 1, сумма которых равна 1. Эти числа часто называют оценками вероятности классов, но их соответствие наблюдаемым частотам нужно отдельно проверять калибровочной кривой или метрикой калибровки. Без такой проверки значение `0.8` нельзя буквально читать как «в 80% подобных случаев будет класс 1».

Покажем различие между сырым линейным значением, результатом `predict_proba` и меткой после стандартного порога.

```python
from sklearn.linear_model import LogisticRegression

X_train = [[-2.0], [-1.0], [1.0], [2.0]]
y_train = [0, 0, 1, 1]

classifier = LogisticRegression(max_iter=1000, random_state=42)
classifier.fit(X_train, y_train)

print(classifier.decision_function([[0.5]]).round(3))
print(classifier.predict_proba([[0.5]]).round(3))
print(classifier.predict([[0.5]]))
```

```text
[0.503]
[[0.377 0.623]]
[1]
```

Сырое значение `0.503` не ограничено интервалом `[0, 1]`. `predict_proba` вернул оценки для классов `0` и `1`; вторая оценка больше `0.5`, поэтому `predict` выбрал класс `1`. Конкретные числа относятся только к этому учебному набору и ничего не говорят о калибровке на новых данных.

Модель даёт сильную и понятную точку отсчёта, быстро обучается и хорошо работает с one-hot-признаками. Линейная граница может быть слишком простой, а величина коэффициента зависит от масштаба и кодирования. Положительный коэффициент означает рост линейного значения модели при прочих равных; он не доказывает причинный эффект или важность в реальном мире.

Мы разделили сырой score, модельную оценку из `predict_proba` и итоговую метку. Если линейная граница систематически ошибается на нелинейных сочетаниях признаков, проверим модель из правил-развилок.

## Дерево и случайный лес

Дерево последовательно задаёт вопросы вида `age <= 12.5`. Каждый вопрос делит текущую группу строк на две ветви; в листе остаётся прогноз для дошедших туда строк. На примере ниже дерево должно выучить простое правило: значения не больше `1` относятся к классу `0`, а значения от `2` — к классу `1`.

```python
from sklearn.tree import DecisionTreeClassifier

X_train = [[0.0], [1.0], [2.0], [3.0]]
y_train = [0, 0, 1, 1]

tree = DecisionTreeClassifier(
    max_depth=1,
    random_state=42,
)
tree.fit(X_train, y_train)

print(tree.tree_.threshold[0])
print(tree.predict([[0.5], [2.5]]))
```

```text
1.5
[0 1]
```

Корневой порог `1.5` действительно отделил две пары обучающих строк, а два новых значения прошли в разные листья. Неглубокое дерево легко показать и объяснить, но глубокое может запомнить отдельные обучающие примеры и переобучиться.

Случайный лес уменьшает зависимость результата от одного дерева: он обучает много различающихся деревьев и объединяет их ответы. В классификации деревья голосуют, в регрессии их числовые прогнозы усредняются. Проверим усреднение на небольшой регрессионной задаче.

```python
from sklearn.ensemble import RandomForestRegressor

X_train = [[0.0], [1.0], [2.0], [3.0]]
y_train = [0.0, 1.0, 4.0, 9.0]

forest = RandomForestRegressor(
    n_estimators=200,
    min_samples_leaf=1,
    random_state=42,
    n_jobs=1,
)
forest.fit(X_train, y_train)

prediction = forest.predict([[2.5]])
print(prediction.shape)
print(prediction.round(2))
```

```text
(1,)
[4.91]
```

Лес вернул массив из одного числового прогноза — среднего ответа 200 деревьев. Значение находится между соседними обучающими целями `4` и `9`, но это не правило интерполяции и на другом `random_state` оно может немного измениться. Лес улавливает нелинейные связи и обычно не требует масштабирования. Он тяжелее, а стандартная оценка важности признаков (feature importance) может вводить в заблуждение и не является причинным объяснением.

Мы увидели одно правило дерева и усреднённый ответ ансамбля. Когда важна гибкая граница между классами в масштабированном пространстве признаков, можно проверить SVM с RBF-ядром.

## SVM с RBF-ядром

SVM ищет границу с большим зазором между классами. **Зазор** — свободная область между границей и ближайшими обучающими объектами разных классов. RBF-ядро позволяет изогнуть эту границу, сравнивая локальную близость объектов, хотя исходные признаки остаются прежними. Масштабирование должно входить в тот же `Pipeline`, чтобы параметры преобразования вычислялись только по обучающим данным.

```python
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVC

X_train = [[-2.0, 0.0], [-1.0, 0.0], [1.0, 0.0], [2.0, 0.0]]
y_train = [0, 0, 1, 1]

svm = make_pipeline(
    StandardScaler(),
    SVC(C=10, gamma="scale", kernel="rbf"),
)
svm.fit(X_train, y_train)

print(svm.predict([[-1.5, 0.0], [1.5, 0.0]]))
```

```text
[0 1]
```

Два новых объекта оказались по разные стороны выученной границы. `C` управляет компромиссом между широким зазором и ошибками на обучающей выборке, а `gamma` задаёт локальность влияния отдельного примера. Эти параметры нельзя выбирать по тестовой выборке.

SVM чувствителен к масштабу признаков и может медленно работать на очень больших наборах. Для Digits он удобен, но это не делает его универсально лучшим алгоритмом.

## Как выбирать

Примеры выше показывают форму результата, но их четыре строки не позволяют сравнивать качество алгоритмов. Не выбирайте модель по значению метрики на обучении или одному удачному разбиению. Зафиксируйте метрику, сравните кандидатов одной процедурой кросс-валидации, проверьте стабильность и ошибки, затем один раз оцените выбранную процедуру на отложенной тестовой выборке. Чем сложнее модель и поиск, тем больше риск подгонки процедуры.
