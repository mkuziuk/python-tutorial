## k ближайших соседей (k-NN)

k-NN не строит явную формулу. Для нового объекта он находит `k` ближайших обучающих объектов и голосует их классами. Слова «вектор», «координата» и «расстояние» разобраны в [математическом словаре](../numpy/#math-vocabulary). Для двух векторов расстояние Евклида — длина разности:

```text
d(x, z) = √Σⱼ(xⱼ − zⱼ)²
```

Здесь `x` и `z` — два объекта с одинаковым набором признаков, индекс `j` перебирает признаки, а `Σ` означает «сложить». Для `x = [1, 2]` и `z = [4, 6]` разности равны `−3` и `−4`, поэтому расстояние равно `√(9 + 16) = 5`. Чем меньше число, тем ближе объекты в выбранном пространстве признаков.

```python
from sklearn.neighbors import KNeighborsClassifier
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler

knn = make_pipeline(
    StandardScaler(),
    KNeighborsClassifier(n_neighbors=5),
)
```

Масштабирование важно: признак с большими числами иначе доминирует в расстоянии. Например, разница в возрасте на 10 лет численно меньше разницы в стоимости билета на 100 единиц и почти исчезнет без общего масштаба. Малое `k` чувствительно к шуму, слишком большое сглаживает полезную локальную структуру.

## Логистическая регрессия

Несмотря на название, это классификатор. Он складывает признаки с весами, преобразует результат в вероятность класса и применяет порог.

До преобразования модель вычисляет линейное значение:

```text
score = b + w₁x₁ + w₂x₂ + …
```

`x₁, x₂` — значения признаков, `w₁, w₂` — коэффициенты, выученные по обучающей выборке, `b` — свободный член. Затем вычисленное значение плавно переводится в число от 0 до 1. Порог 0,5 — лишь возможное рабочее правило, а не часть истины о классе.

```python
from sklearn.linear_model import LogisticRegression

classifier = LogisticRegression(max_iter=1000, random_state=42)
```

Модель даёт сильную и понятную точку отсчёта, быстро обучается и хорошо работает с one-hot-признаками. Линейная граница может быть слишком простой, а величина коэффициента зависит от масштаба и кодирования. Положительный коэффициент означает рост значения модели при прочих равных; он не доказывает причинный эффект или важность в реальном мире.

## Дерево и случайный лес

Дерево последовательно задаёт вопросы вида `age <= 12.5`. Каждый вопрос делит текущую группу строк на две ветви; в листе остаётся прогноз для дошедших туда строк. Неглубокое дерево легко показать и объяснить, но глубокое может запомнить отдельные обучающие примеры и переобучиться.

```python
from sklearn.tree import DecisionTreeClassifier

tree = DecisionTreeClassifier(
    max_depth=4,
    min_samples_leaf=10,
    random_state=42,
)
```

Случайный лес обучает много различающихся деревьев и объединяет их ответы: в классификации деревья голосуют, в регрессии их числовые прогнозы усредняются.

```python
from sklearn.ensemble import RandomForestRegressor

forest = RandomForestRegressor(
    n_estimators=200,
    min_samples_leaf=3,
    random_state=42,
    n_jobs=1,
)
```

Лес улавливает нелинейные связи и обычно не требует масштабирования. Он тяжелее, а стандартная оценка важности признаков (feature importance) может вводить в заблуждение и не является причинным объяснением.

## SVM с RBF-ядром

SVM ищет границу с большим зазором между классами. **Зазор** — свободная область между границей и ближайшими обучающими объектами разных классов. RBF-ядро позволяет изогнуть эту границу, сравнивая локальную близость объектов, хотя исходные признаки остаются прежними.

```python
from sklearn.svm import SVC

svm = SVC(C=10, gamma="scale", kernel="rbf")
```

- `C` управляет компромиссом между широким зазором и ошибками на обучающей выборке;
- `gamma` задаёт локальность влияния отдельного примера.

SVM чувствителен к масштабу признаков и может медленно работать на очень больших наборах. Для Digits он удобен, но это не делает его универсально лучшим алгоритмом.

## Как выбирать

Не выбирайте модель по значению метрики на обучении или одному удачному разбиению. Зафиксируйте метрику, сравните кандидатов одной процедурой кросс-валидации, проверьте стабильность и ошибки, затем один раз оцените модель на отложенной тестовой выборке. Чем сложнее модель и поиск, тем больше риск подгонки процедуры.
