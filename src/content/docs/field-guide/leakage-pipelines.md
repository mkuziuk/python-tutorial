## Что такое утечка

Утечка возникает, когда при обучении используется информация, которой не будет во время реального прогноза или которая относится к проверочной выборке. Из-за этого оценка становится слишком оптимистичной.

Три частых вида:

1. **Утечка из будущего (post-outcome leakage)** — признак появился после предсказываемого события (`boat`, `body` в Titanic).
2. **Утечка при подготовке данных (preprocessing leakage)** — медиана, масштаб или набор признаков рассчитаны по всей таблице до разбиения.
3. **Утечка между группами (group leakage)** — связанные записи одного человека или источника оказались и в обучающей, и в тестовой выборке.

Небольшой пример утечки при подготовке данных: в обучающей выборке известны возраста `[20, 40]`, а в тестовой — `[100]`. Медиана обучающей выборки равна 30. Если сначала объединить все строки, медиана станет 40: тестовое значение уже повлияло на правило заполнения пропусков. На большой таблице такая утечка менее заметна, но принцип остаётся тем же.

## Почему помогает `Pipeline`

`Pipeline.fit(X_train, y_train)` обучает каждый шаг подготовки данных только на обучающей выборке. При `predict(X_test)` конвейер применяет сохранённые параметры без переобучения.

```python
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier

model = make_pipeline(
    StandardScaler(),
    KNeighborsClassifier(n_neighbors=5),
)
# Параметры масштабирования вычисляются только по X_train.
model.fit(X_train, y_train)
predictions = model.predict(X_test)
```

## Разные типы столбцов

```python
from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import OneHotEncoder, StandardScaler

numeric = make_pipeline(
    SimpleImputer(strategy="median"),
    StandardScaler(),
)
categorical = make_pipeline(
    SimpleImputer(strategy="most_frequent"),
    OneHotEncoder(handle_unknown="ignore"),
)

preprocess = ColumnTransformer([
    ("num", numeric, numeric_columns),
    ("cat", categorical, categorical_columns),
])
```

`SimpleImputer` учит значение для заполнения пропусков. `OneHotEncoder` превращает категорию, например `embarked="C"`, в отдельные 0/1-столбцы. `StandardScaler` сохраняет среднее и масштаб числовых признаков. Все эти величины являются обучаемыми параметрами подготовки данных, а не нейтральной «уборкой» таблицы.

Добавьте модель последним шагом. Тогда тот же объект можно безопасно использовать при кросс-валидации: для каждой части разбиения заполнение пропусков, кодирование, масштабирование и модель будут заново обучаться только на соответствующей обучающей части.

## Чего `Pipeline` не решает

`Pipeline` не знает смысла признака. Он не удалит `boat`, не выберет правильные группы, не изолирует партию C и не сформулирует сценарий внедрения. Смысловую границу задаёт исследователь.

## Проверочный список

- Сначала разбиение, потом `.fit()` любых преобразований.
- Цель не входит в `X`, даже под другим именем.
- Каждый признак существовал в момент прогноза.
- Связанные объекты не пересекают границу оценки.
- Тестовая выборка не влияет на признаки, модель, гиперпараметры или порог.
- В кросс-валидацию передаётся целый конвейер, а не заранее преобразованная матрица.

## Типичные ловушки

- `scaler.fit_transform(X)` до разбиения.
- Заполнение пропусков медианой всей таблицы.
- Выбор «лучших» столбцов по корреляции с `y_test`.
- Сохранёние идентификатора, который кодирует цель или источник.
- Вызов `fit_transform()` на тестовой выборке вместо `transform()`.
