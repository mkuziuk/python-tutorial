## Что это

`DataFrame` — таблица со строками-наблюдениями, именованными столбцами и индексом. `Series` — один столбец. Имена помогают аудировать признаки до того, как таблица превратится в матрицу модели.

Если слова «матрица», «вектор» или `shape` пока непривычны, сначала откройте [математический словарь Части II](../numpy/#math-vocabulary).

Например, одна строка может описывать пассажира, а столбцы — известные о нём поля:

```python
import pandas as pd

df = pd.DataFrame({
    "age": [22.0, None, 35.0],
    "sex": ["female", "male", "female"],
    "survived": [1, 0, 1],
})

print(df.shape)           # (3, 3): три строки, три столбца
print(df["age"].isna())  # False, True, False
```

Индекс `0, 1, 2` помогает pandas выравнивать строки, но сам по себе не обязан быть признаком модели. `None` в числовом столбце превращается в пропуск, а не в возраст 0.

Для готового снимка данных сначала откройте терминал прямо в папке проекта: в системном файловом приложении найдите распакованную папку расследования, щёлкните по ней правой кнопкой и выберите команду вроде **«Открыть в Терминале»**. Название зависит от системы. Тогда относительный путь `data/titanic.csv` отсчитывается от нужной папки, и вводить непривычную команду `cd` не требуется.

Первый вопрос к файлу — прочитался ли он как таблица ожидаемой формы. Вход — `data/titanic.csv`; результат — число строк и столбцов и первые пять строк:

```python
import pandas as pd

df = pd.read_csv("data/titanic.csv")
print(df.shape)
df.head()
```

В Jupyter последняя строка показывает таблицу. Например, форма `(891, 12)` читается как «891 пассажир и 12 полей», а не просто как два числа. Конкретная форма зависит от снимка данных; если путь неверен, `read_csv` остановится с `FileNotFoundError`, и анализ продолжать нельзя. После проверки загрузки можно перейти к типам, пропускам и числу разных значений.

## Первый аудит

Не начинайте с обучения. Сначала ответьте на четыре вопроса. Чтобы результат был воспроизводимым, ниже используется маленькая самостоятельная таблица: три пассажира, один пропуск возраста и два класса каюты. Ожидаемый аудит должен показать форму `(3, 4)`, типы столбцов, долю пропусков `1/3` для `age` и число разных значений:

```python
import pandas as pd

df = pd.DataFrame({
    "passenger_id": [101, 102, 103],
    "age": [22.0, None, 35.0],
    "sex": ["female", "male", "female"],
    "pclass": [3, 1, 3],
})

print("shape:", df.shape)
print("dtypes:")
print(df.dtypes)
print("missing share:")
print(df.isna().mean().sort_values(ascending=False))
print("unique including missing:")
print(df.nunique(dropna=False).sort_values())
```

```text
shape: (3, 4)
dtypes:
passenger_id      int64
age             float64
sex              object
pclass            int64
missing share:
age             0.333333
passenger_id    0.000000
sex             0.000000
pclass          0.000000
unique including missing:
sex             2
pclass          2
age             3
passenger_id    3
```

Форма подтверждает единицу таблицы: каждая из трёх строк описывает одного пассажира. `age` хранится как `float64`, потому что числовой столбец содержит пропуск; `0.333333` буквально означает один пропуск среди трёх строк. `passenger_id` уникален в каждой строке и похож на идентификатор, а `sex` и `pclass` имеют по два разных значения и могут быть категориальными признаками. На малой таблице кардинальность ненадёжна, поэтому назначение столбца всё равно сверяют с описанием источника.

Итак, аудит установил форму, типы, пропуски и кандидата в идентификаторы. Следующий выбор — явно отделить допустимые признаки от цели, не включая `passenger_id` автоматически.

Тип `object` не гарантирует, что столбец — полезная категория. Это может быть имя, свободный текст, смешанный формат или число, прочитанное как строка.

## Признаки и цель

```python
import pandas as pd

df = pd.DataFrame({
    "passenger_id": [101, 102, 103],
    "pclass": [3, 1, 3],
    "sex": ["female", "male", "female"],
    "age": [22.0, None, 35.0],
    "fare": [7.25, 71.28, 8.05],
    "embarked": ["S", "C", "S"],
    "survived": [1, 0, 1],
})

target = "survived"
features = ["pclass", "sex", "age", "fare", "embarked"]

X = df[features].copy()
y = df[target].copy()

print(X.shape, y.shape)
```

Вывод `(3, 5) (3,)` означает три объекта с пятью признаками и три соответствующих ответа. Явный список безопаснее, чем «все столбцы, кроме цели»: так `passenger_id` и post-outcome поля не попадут в модель случайно. Список ещё не доказывает, что признаки были доступны в момент прогноза; это проверяют по описанию данных до обучения.

## Пропуски

Пропуск — не ноль и не пустая строка. Его обработка зависит от смысла:

```python
import pandas as pd

df = pd.DataFrame({
    "pclass": [1, 1, 3, 3],
    "age": [38.0, None, 22.0, 35.0],
})

print("missing:", df["age"].isna().sum())
print(df.groupby("pclass", dropna=False)["age"].median())
```

Результат показывает один пропуск; медианы известных возрастов равны `38.0` для первого класса и `28.5` для третьего. Это описание четырёх строк, а не правило заполнения: малые группы нестабильны, а причина пропуска неизвестна. В Части II пропуски заполняет `SimpleImputer` внутри `Pipeline`. Если вычислить медиану по всей таблице до разбиения, информация из тестовой выборки повлияет на подготовку данных.

## Групповые сводки

```python
import pandas as pd

df = pd.DataFrame({
    "pclass": [1, 1, 3, 3],
    "survived": [1, 0, 1, 1],
})

summary = (
    df.groupby("pclass", dropna=False)
      .agg(rows=("survived", "size"), survival_rate=("survived", "mean"))
      .reset_index()
)
print(summary)
```

```text
   pclass  rows  survival_rate
0       1     2            0.5
1       3     2            1.0
```

Если `survived` кодируется как 0/1, его среднее равно доле единиц. Здесь первый класс имеет одну единицу из двух, а третий — две из двух. В обеих группах лишь по две строки, поэтому разница описывает учебную таблицу, но не доказывает устойчивую связь и тем более причинное влияние класса. Всегда показывайте `rows` рядом с долей; следующим шагом будет проверить срез на полной выборке и оценить неопределённость.

## Типичные ловушки

- Интерпретировать индекс как признак.
- Терять соответствие `X` и `y` после раздельной сортировки или удаления строк.
- Делать chained assignment и не понимать, изменилась ли исходная таблица.
- Заменять все пропуски одним значением до разбиения.
- Считать корреляцию доказательством доступности или причинности.
