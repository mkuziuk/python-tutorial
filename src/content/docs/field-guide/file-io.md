## Текст или байты

Текстовый файл Python превращает в `str`; для русского текста явно указывайте UTF-8:

```python
from pathlib import Path

path = Path("data") / "anonymous.txt"
text = path.read_text(encoding="utf-8")
```

Бинарный режим возвращает `bytes` без декодирования. Он нужен для хешей, изображений, архивов и парсера `.eml`, которому важны исходные байты:

```python
raw = path.read_bytes()
```

## Контекстный менеджер `with`

Для пошагового чтения откройте файл через `with`. Python закроет его и при успешном завершении, и при исключении:

```python
with path.open("r", encoding="utf-8") as file:
    first_line = file.readline()
```

Основные режимы:

| Режим | Значение |
| --- | --- |
| `"r"` | читать текст |
| `"w"` | перезаписать текст |
| `"a"` | дописать текст в конец |
| `"rb"` | читать байты |
| `"wb"` | перезаписать байты |

Режимы `w` и `wb` очищают существующий файл перед записью. Для готовой строки проще использовать `path.write_text(text, encoding="utf-8")`.

## Чтение блоками

Хеш большого файла не нужно загружать целиком. Читайте фиксированные блоки до конца файла:

```python
import hashlib

digest = hashlib.sha256()
with path.open("rb") as file:
    while chunk := file.read(64 * 1024):
        digest.update(chunk)

print(digest.hexdigest())
```

`file.read()` возвращает пустые байты `b""` в конце файла; они считаются ложным значением и останавливают `while`.

## Ошибки файловой системы

Чтение может завершиться `FileNotFoundError`, `PermissionError`, `UnicodeDecodeError` или более общим `OSError`. Ловите исключение только там, где можете добавить понятный контекст или выбрать безопасное действие:

```python
try:
    text = path.read_text(encoding="utf-8")
except OSError as exc:
    # from exc сохраняет OSError как исходную причину нового RuntimeError.
    raise RuntimeError(f"Не удалось прочитать {path}") from exc
```

## Типичные ловушки

- Относительный путь считается от текущей рабочей папки, а не от файла с кодом.
- Не смешивайте `str` и `bytes`: сначала осознанно декодируйте байты или кодируйте строку.
- Не полагайтесь на системную кодировку для учебных данных.
- Перед перезаписью важного файла подготовьте данные полностью; ошибка посередине не должна оставлять незаметно поврежденный результат.
