## Что это

Включение создаёт новую коллекцию из элементов другой коллекции. Оно объединяет выражение, цикл и необязательное условие.

Возьмём один конкретный вход — список из трёх строк с разным регистром.
Обычный цикл создаёт новый список нормализованных слов:

```python
words = ["След", "Пауза", "АРХИВ"]
normalized = []
for word in words:
    normalized.append(word.casefold())

print(normalized)
```

```text
['след', 'пауза', 'архив']
```

`normalized` имеет форму «список строк» и сохраняет порядок `words`.
Списковое включение создаёт то же значение короче:

То же списковым включением:

```python
words = ["След", "Пауза", "АРХИВ"]
normalized = [word.casefold() for word in words]
print(normalized)
```

```text
['след', 'пауза', 'архив']
```

Совпадающий вывод подтверждает, что изменилась запись преобразования, а не
форма результата. Читайте включение слева направо: «взять `word.casefold()`
для каждого `word` из `words`». Если нужно одновременно отфильтровать элементы
или получить другой контейнер, меняются условие и скобки.

## Список, множество и словарь

На том же входе сравним три формы результата:

```python
words = ["След", "пауза", "следопыт"]
long_words = [word for word in words if len(word) > 5]
unique_words = {word.casefold() for word in words}
length_by_word = {word: len(word) for word in words}

print(long_words)
print(sorted(unique_words))
print(length_by_word)
```

```text
['следопыт']
['пауза', 'след', 'следопыт']
{'След': 4, 'пауза': 5, 'следопыт': 8}
```

- `[...]` создаёт список и сохраняет порядок;
- `{...}` создаёт множество уникальных значений;
- `{key: value ...}` создаёт словарь.

Вывод показывает конкретную форму каждого контейнера: фильтр оставил одну
длинную строку, множество хранит нормализованные уникальные слова, а словарь
связывает исходное написание с длиной. `sorted()` применён только для
стабильного показа множества. Часть `if` отбрасывает элементы, но не заменяет
обычное ветвление, когда действий несколько. Если готовый контейнер вообще не
нужен, значения можно вычислять по одному.

## Генераторное выражение

Круглые скобки создают не готовый список, а поток значений, который вычисляется по мере чтения:

```python
words = ["след", "пауза", ""]
total_letters = sum(len(word) for word in words)
has_empty = any(not word for word in words)

print(total_letters)
print(has_empty)
```

```text
9
True
```

`sum()` сложил длины трёх строк, а `any()` остановился, найдя пустую строку.
Генератор не хранит отдельный список длин или флагов, потому что обе функции
читают поток один раз. Когда нужно материализовать все сочетания, создают
список.

Генератор удобен для `sum()`, `any()`, `all()`, `tuple()` и других функций, которым достаточно пройти по значениям один раз. Если результат нужно читать несколько раз, индексировать или изменять, создайте список.

## Вложенные выражения

Несколько `for` читаются в том же порядке, что и вложенные циклы:

```python
names = ["draft.txt", "notes.txt", "report.txt"]
pairs = [
    (left, right)
    for left in names
    for right in names
    if left < right
]
print(pairs)
```

```text
[('draft.txt', 'notes.txt'), ('draft.txt', 'report.txt'), ('notes.txt', 'report.txt')]
```

Три имени дали три неупорядоченные пары без повторов вида `(a, a)` и без
зеркальных вариантов. Условие `left < right` использует лексикографический
порядок строк только как способ оставить один вариант каждой пары.

Если выражение перестает помещаться в голове, разверните его в обычные циклы и дайте промежуточным шагам имена.

## Типичные ловушки

- Включение всегда создаёт новую коллекцию; оно не меняет исходную.
- Фигурные скобки без `key: value` создают множество, а не словарь.
- Генератор расходуется после одного полного прохода.
- Не прячьте обработку ошибок и побочные эффекты внутри длинного включения.
