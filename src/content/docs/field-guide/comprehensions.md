## Что это

Включение создаёт новую коллекцию из элементов другой коллекции. Оно объединяет выражение, цикл и необязательное условие.

Обычный цикл:

```python
normalized = []
for word in words:
    normalized.append(word.casefold())
```

То же списковым включением:

```python
normalized = [word.casefold() for word in words]
```

Читайте его слева направо: «взять `word.casefold()` для каждого `word` из `words`».

## Список, множество и словарь

Тип скобок определяет результат:

```python
long_words = [word for word in words if len(word) > 5]
unique_words = {word.casefold() for word in words}
length_by_word = {word: len(word) for word in words}
```

- `[...]` создаёт список и сохраняет порядок;
- `{...}` создаёт множество уникальных значений;
- `{key: value ...}` создаёт словарь.

Часть `if` отбрасывает элементы, но не заменяет обычное ветвление, когда действий несколько.

## Генераторное выражение

Круглые скобки создают не готовый список, а поток значений, который вычисляется по мере чтения:

```python
total_letters = sum(len(word) for word in words)
has_empty = any(not word for word in words)
```

Генератор удобен для `sum()`, `any()`, `all()`, `tuple()` и других функций, которым достаточно пройти по значениям один раз. Если результат нужно читать несколько раз, индексировать или изменять, создайте список.

## Вложенные выражения

Несколько `for` читаются в том же порядке, что и вложенные циклы:

```python
pairs = [
    (left, right)
    for left in names
    for right in names
    if left < right
]
```

Если выражение перестает помещаться в голове, разверните его в обычные циклы и дайте промежуточным шагам имена.

## Типичные ловушки

- Включение всегда создаёт новую коллекцию; оно не меняет исходную.
- Фигурные скобки без `key: value` создают множество, а не словарь.
- Генератор расходуется после одного полного прохода.
- Не прячьте обработку ошибок и побочные эффекты внутри длинного включения.
