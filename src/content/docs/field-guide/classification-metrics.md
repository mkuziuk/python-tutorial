## Матрица ошибок

Для выбранного «положительного» класса прогноз даёт четыре исхода:

| | Предсказан 0 | Предсказан 1 |
| --- | ---: | ---: |
| Истинный 0 | TN | FP |
| Истинный 1 | FN | TP |

Порядок строк и столбцов проверяйте по подписи библиотеки. «Положительный» не обязательно означает хороший исход — это просто класс, для которого считаются показатели.

## Precision, recall и F1

```text
precision = TP / (TP + FP)
recall    = TP / (TP + FN)
```

Precision спрашивает: «какая доля отмеченных объектов действительно положительна?» Recall: «какую долю всех положительных объектов мы нашли?»

```text
F1 = 2 × precision × recall / (precision + recall)
```

F1 объединяет precision и recall, но не учитывает реальную стоимость ошибок. Сначала формулируйте операционное ограничение, затем выбирайте метрику.

Например, среди 100 объектов получены `TP=30`, `FP=10`, `FN=5`, `TN=55`:

- accuracy равна `(30 + 55) / 100 = 0.85`;
- precision равна `30 / (30 + 10) = 0.75`: четверть отметок оказалась ложной;
- recall равна `30 / (30 + 5) ≈ 0.86`: найдено 30 из 35 положительных;
- F1 равна примерно `0.80`.

Одна и та же accuracy могла бы получиться при другом соотношении FP и FN. Поэтому рядом с процентом всегда полезно показать исходные количества и знаменатель.

```python
from sklearn.metrics import classification_report, confusion_matrix

print(confusion_matrix(y_test, predictions))
print(classification_report(y_test, predictions, digits=3))
```

## Порог

Вероятность становится меткой после сравнения с порогом. Если сами термины пока непривычны, см. [«вероятность и порог» в математическом словаре](../numpy/#math-vocabulary):

```python
probability = model.predict_proba(X_validation)[:, 1]
predictions = (probability >= threshold).astype(int)
```

Понижение порога обычно повышает recall и число ложноположительных решений; повышение — наоборот. Выбирайте порог по проверочной выборке или вневыборочным прогнозам, а не по финальной тестовой выборке.

## Много классов

Для десяти цифр F1 можно посчитать отдельно, затем усреднить:

- **macro-F1** даёт каждому классу одинаковый вес;
- **weighted-F1** взвешивает по числу объектов класса;
- **micro-F1** агрегирует решения всех классов.

Macro-F1 полезен, когда слабый класс не должен раствориться в общем результате.

Если F1 трёх классов равны `0.95`, `0.60` и `0.85`, macro-F1 — их обычное среднее: `(0.95 + 0.60 + 0.85) / 3 = 0.80`. У каждого класса один голос, даже если строк первого класса в десять раз больше.

```python
from sklearn.metrics import f1_score

macro_f1 = f1_score(y_test, predictions, average="macro")
```

## Срезы ошибок

Общая метрика может скрывать регион, класс билета, партию или отдельную цифру. Для каждого среза показывайте:

- размер `n`;
- число положительных/классов;
- ту же метрику и ошибки;
- неопределенность или предупреждение о малом `n`.

Срез не становится доказательством дискриминации или причины. Это диагностический сигнал для следующей проверки.

## Типичные ловушки

- Считать только долю правильных ответов при сильном дисбалансе и не сравнивать с базовой моделью.
- Выбирать порог по тестовой выборке.
- Показывать процент без знаменателя.
- Сравнивать группы с разной базовой долей без контекста.
- Называть `predict_proba` хорошо откалиброванной вероятностью без проверки калибровки.
