## Матрица ошибок

Для выбранного «положительного» класса прогноз даёт четыре исхода:

| | Предсказан 0 | Предсказан 1 |
| --- | ---: | ---: |
| Истинный 0 | TN | FP |
| Истинный 1 | FN | TP |

Порядок строк и столбцов проверяйте по подписи библиотеки. «Положительный» не обязательно означает хороший исход — это просто класс, для которого считаются показатели.

## Precision, recall и F1

Сначала зафиксируйте класс `1` как положительный и цену двух ошибок. `FP` —
ложная тревога: истинный класс `0`, а прогноз `1`. `FN` — пропуск: истинный
класс `1`, а прогноз `0`. Обе метрики лежат от 0 до 1, и большее значение
лучше, но знаменатели у них разные:

```text
precision = TP / (TP + FP)
recall    = TP / (TP + FN)
```

Precision спрашивает: «какая доля отмеченных объектов действительно положительна?» Recall: «какую долю всех положительных объектов мы нашли?»

```text
F1 = 2 × precision × recall / (precision + recall)
```

F1 объединяет precision и recall, но не учитывает реальную стоимость ошибок. Сначала формулируйте операционное ограничение, затем выбирайте метрику.

Например, среди 100 объектов получены `TP=30`, `FP=10`, `FN=5`, `TN=55`:

- accuracy равна `(30 + 55) / 100 = 0.85`;
- precision равна `30 / (30 + 10) = 0.75`: четверть отметок оказалась ложной;
- recall равна `30 / (30 + 5) ≈ 0.86`: найдено 30 из 35 положительных;
- F1 равна примерно `0.80`.

Одна и та же accuracy могла бы получиться при другом соотношении FP и FN. Поэтому рядом с процентом всегда полезно показать исходные количества и знаменатель.

Следующий автономный пример восстанавливает те же четыре количества из меток.
В `confusion_matrix` строки соответствуют истинным классам, столбцы —
предсказанным, а явный `labels=[0, 1]` фиксирует их порядок:

```python
from sklearn.metrics import confusion_matrix, precision_recall_fscore_support

y_true = [1] * 30 + [0] * 10 + [1] * 5 + [0] * 55
y_pred = [1] * 40 + [0] * 60

print(confusion_matrix(y_true, y_pred, labels=[0, 1]))
print(precision_recall_fscore_support(
    y_true, y_pred, average="binary", zero_division=0
))
```

```text
[[55 10]
 [ 5 30]]
(0.75, 0.8571428571428571, 0.8, None)
```

Первая строка матрицы подтверждает 55 верных отрицательных решений и 10 ложных
тревог; вторая — 5 пропусков и 30 найденных положительных объектов. Кортеж
возвращает `(precision, recall, F1, support)`; при `average="binary"` support
не возвращается, поэтому последнее значение равно `None`. Мы проверили
числители и знаменатели, а уже затем можем выбирать порог под цену FP и FN.

## Порог

`predict_proba` возвращает модельную оценку от 0 до 1 для каждого класса.
После проверки калибровки её можно интерпретировать как вероятность; без такой
проверки это прежде всего величина для ранжирования и выбора порога. Порог
превращает оценку в метку. Если термины пока непривычны, см.
[«вероятность и порог» в математическом словаре](../numpy/#math-vocabulary):

Проверим порог `0.70` на самостоятельном примере. Модель обучается на четырёх
одномерных объектах, а затем возвращает оценки для трёх новых:

```python
import numpy as np
from sklearn.linear_model import LogisticRegression

X_train = np.array([[-2.0], [-1.0], [1.0], [2.0]])
y_train = np.array([0, 0, 1, 1])
X_validation = np.array([[-0.5], [0.5], [1.5]])

model = LogisticRegression(max_iter=1000, random_state=42)
model.fit(X_train, y_train)

threshold = 0.70
class_1_column = np.flatnonzero(model.classes_ == 1)[0]
class_1_score = model.predict_proba(X_validation)[:, class_1_column]
predictions = (class_1_score >= threshold).astype(int)

print(np.round(class_1_score, 3))
print(predictions)
```

```text
[0.377 0.623 0.819]
[0 0 1]
```

`model.classes_` связывает индекс столбца с классом `1`, поэтому код не
полагается на неявное положение класса. При пороге `0.70` только оценка `0.819`
становится меткой `1`; оценка `0.623` осталась меткой `0`, хотя при стандартном
пороге `0.50` дала бы другой результат. Понижение порога обычно повышает recall и число
ложноположительных решений; повышение — наоборот. Граница `>=` означает, что
значение, точно равное порогу, относится к классу `1`. Выбирайте порог по
проверочной выборке или вневыборочным прогнозам, а не по финальной тестовой
выборке.

## Много классов

Для десяти цифр F1 можно посчитать отдельно, затем усреднить:

- **macro-F1** даёт каждому классу одинаковый вес;
- **weighted-F1** взвешивает по числу объектов класса;
- **micro-F1** агрегирует решения всех классов.

Macro-F1 полезен, когда слабый класс не должен раствориться в общем результате.

Если F1 трёх классов равны `0.95`, `0.60` и `0.85`, macro-F1 — их обычное среднее: `(0.95 + 0.60 + 0.85) / 3 = 0.80`. У каждого класса один голос, даже если строк первого класса в десять раз больше.

```python
from sklearn.metrics import f1_score

per_class_f1 = [0.95, 0.60, 0.85]
macro_f1 = sum(per_class_f1) / len(per_class_f1)
print(f"{macro_f1:.2f}")
```

```text
0.80
```

Число `0.80` — среднее по трём классам, а не доля правильных ответов. Оно
показывает, что слабый второй класс влияет на итог так же сильно, как первый.
Для реальных меток используйте
`f1_score(y_true, predictions, average="macro")` и рядом показывайте F1 и
число объектов каждого класса.

## Срезы ошибок

Общая метрика может скрывать регион, класс билета, партию или отдельную цифру. Для каждого среза показывайте:

- размер `n`;
- число положительных/классов;
- ту же метрику и ошибки;
- неопределенность или предупреждение о малом `n`.

Срез не становится доказательством дискриминации или причины. Это диагностический сигнал для следующей проверки.

## Типичные ловушки

- Считать только долю правильных ответов при сильном дисбалансе и не сравнивать с базовой моделью.
- Выбирать порог по тестовой выборке.
- Показывать процент без знаменателя.
- Сравнивать группы с разной базовой долей без контекста.
- Называть `predict_proba` хорошо откалиброванной вероятностью без проверки калибровки.
