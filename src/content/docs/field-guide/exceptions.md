## Что это

Исключение прерывает текущую операцию. Например, `ip_address()` вызывает `ValueError` для некорректного адреса. `except ValueError` обрабатывает этот случай, после чего программа может продолжить работу.

## Когда использовать

Используйте `try` / `except`, когда ошибка ожидаема и у программы есть понятное действие: пропустить запись, добавить предупреждение, показать аккуратный отчёт.

```python
# Сначала Python выполняет блок try.
try:
    number = int("17")
except ValueError:
    # Этот блок выполняется, только если int() бросил ValueError.
    number = 0
```

## Мини-пример

```python
from pathlib import Path

try:
    text = Path("missing.txt").read_text(encoding="utf-8")
except FileNotFoundError:
    # После совпавшего except выполнение продолжается со следующей строки после конструкции.
    text = ""
```

## Ещё примеры

Ловите конкретную ожидаемую ошибку как можно ближе к операции, которая её вызывает:

```python
import json
from pathlib import Path

try:
    record = json.loads(Path("record.json").read_text(encoding="utf-8"))
except FileNotFoundError:
    print("файл record.json не найден")
except json.JSONDecodeError as error:
    # as error сохраняет объект исключения, поэтому доступна позиция ошибки error.pos.
    print(f"JSON повреждён около символа {error.pos}")
else:
    # else выполняется, только если блок try завершился без исключения.
    print(f"загружено полей: {len(record)}")
```

`else` выполняется только если исключения не было. Это помогает не помещать дальнейшую обработку в слишком широкий блок `try`.

## Типичные ловушки

- Не ловите голый `except:` без причины: он прячет даже неожиданные ошибки.
- Не продолжайте вычисление с выдуманным значением по умолчанию, если ошибка делает результат недостоверным.
- Не используйте исключения вместо обычной проверки, если проверка простая и ясная.
- В отчёте полезнее показать понятное предупреждение, чем полный traceback.
