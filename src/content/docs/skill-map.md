Карта нужна для ориентации, но не диктует порядок чтения. Основной путь - дела: теория появляется тогда, когда инструменту нужен новый прием.

| Навык | Где появляется | Зачем нужен |
| --- | --- | --- |
| `str` | Дело 01 | Хранить, очищать и сравнивать текст |
| `list` | Дело 01 | Сохранять последовательность слов и результатов |
| `dict` | Дела 01, 02, 05 | Собрать профиль, индекс или запись дела |
| `set` | Дела 01, 02 | Сравнить уникальные признаки и пересечения |
| regex | Дела 01, 03 | Достать слова, ссылки и текстовые сигналы |
| `Counter` | Дела 01, 02 | Посчитать частоты слов, знаков и n-грамм |
| [Функции](./field-guide/functions/) | Дела 01-05 | Разделить инструмент на проверяемые шаги |
| `tuple` | Дело 02 | Представить n-грамму как неизменяемый признак |
| Сортировка | Дело 02 | Построить рейтинг подозрительных совпадений |
| `email` | Дело 03 | Разобрать структуру письма и заголовки |
| `urllib.parse` | Дело 03 | Аккуратно разобрать ссылки |
| `ipaddress` | Дело 03 | Отличить IP-адрес от обычного домена |
| Исключения | Дела 03, 04 | Обработать плохие данные без падения программы |
| `pathlib` | Дела 04, 05 | Работать с путями переносимо и понятно |
| Обход файлов | Дело 04 | Собрать индекс папки |
| `hashlib` | Дело 04 | Найти дубли и изменения по SHA-256 |
| JSON | Дела 04, 05 | Сохранить manifest и состояние системы |
| Классы | Дело 05 | Описать дело и улику как собственные объекты |
| `dataclass` | Дело 05 | Сделать компактные классы данных |
| Композиция | Дело 05 | Собрать систему из маленьких объектов |
| Rich | Дела 01-05 | Показать финальный отчет аккуратной таблицей |

## Путь первой арки

1. [Дело 01](./cases/anonymous-letter/) - текстовый профиль и первый рабочий отчет.
2. [Дело 02](./cases/copy-paste-detector/) - сравнение отчетов через n-граммы.
3. [Дело 03](./cases/phishing-email/) - защитный разбор ловушек в почте и объяснимые правила.
4. [Дело 04](./cases/secret-folder-archive/) - ночной сигнал архива, хеши и manifest.
5. [Дело 05](./cases/investigation-system/) - объектная модель доски расследования.

## Что получится

После пяти дел ученик умеет не только писать отдельные функции, но и собирать небольшие инструменты: читать файлы, превращать данные в структуры, объяснять результат, сохранять состояние и держать код в понятных модулях.
