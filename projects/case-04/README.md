# Расследование 04. Последнее доказательство

Финальный проект собирает выводы I-01–I-03, добавляет последнюю проверку журналов и определяет человека, чьё досье нужно изучить подробно.

## Подготовка

Нужен Python 3.13 или новее. Внешних библиотек нет.

Сначала откройте папку проекта в системном файловом приложении. В Windows
щёлкните по пустому месту папки правой кнопкой мыши и выберите
«Открыть в терминале». В macOS откройте папку в Finder, щёлкните по ней правой
кнопкой и выберите «Службы → Новый терминал в папке». В Linux аналогичная команда
обычно называется «Открыть в терминале». Так терминал сразу начнёт работу в
нужной папке, и вводить `cd` не придётся.

### Windows PowerShell

```powershell
py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1
```

### macOS или Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
```

После активации в начале строки терминала появится `(.venv)`. Это означает,
что команды Python будут использовать окружение именно этого проекта.
Устанавливать зависимости не нужно: `requirements.txt` лишь фиксирует, что
проект использует стандартную библиотеку.

## Запуск

```bash
python final_evidence.py
```

`final_evidence.py` — пустой стартовый файл. Переносите в него листинги главы по порядку. Готовая программа создаст `artifacts/04-investigation-summary.json`.

## Что изучаем

- чтение и проверку JSON;
- поиск записи по стабильному идентификатору;
- объединение выводов с сохранением происхождения и ограничений;
- словарный индекс `person_id → dossier`;
- объяснимый рейтинг и сортировку;
- сохранение итогового отчёта.

Компьютерный результат выбирает направление углублённой проверки. Развязка изложена в конце учебной главы после проверки программы.
