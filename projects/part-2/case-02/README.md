# Дело II-02. Пассажиры после факта

Поставщик показывает почти безошибочный прогноз выживания пассажиров «Титаника». Бюро должно выяснить, какие поля действительно существовали в момент прогноза, а какие появились только после исхода.

## Результат дела

За 4–5 часов вы:

- исследуете типы, пропуски и момент доступности полей;
- распознаете `boat` и `body` как утечку после исхода;
- один раз заморозите внешний stratified holdout;
- соберёте `ColumnTransformer` с импутацией и one-hot кодированием;
- сравните dummy-, честную и заведомо протекающую модели;
- сформулируете границы доказанного.

## Подготовка

Нужен Python 3.12 или 3.13.

```bash
python3 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
python -m pip install -r requirements.txt
jupyter lab case-02.ipynb
```

В Colab используется тот же файл. Bootstrap-ячейка скачивает ZIP дела, сверяет его SHA-256 и работает с вложенным `data/titanic.csv` без ручной правки путей.

## Файлы и происхождение

- `case-02.ipynb` — рабочая тетрадь;
- `solution/case-02-solution.ipynb` — эталон;
- `data/titanic.csv` — замороженный OpenML Titanic, dataset 40945, version 1;
- `data/SOURCE.md`, `data/LICENSE.txt`, `data/CHECKSUMS.sha256` — источник, лицензионная метка и контроль целостности;
- `check_result.md` — качественные и численные ориентиры.

Не заменяйте CSV свежей загрузкой: воспроизводимость аудита зависит от точной копии.
