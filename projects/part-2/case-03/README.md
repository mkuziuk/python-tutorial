# Дело II-03. Цена одной ошибки

После устранения утечки у Бюро остаётся корректно построенная модель, но поставщик продолжает показывать только accuracy. В этом деле вы определите цену ложных решений, выберете порог по операционному правилу и проверите различия между группами.

## Результат дела

За 4–5 часов вы:

- независимо восстановите зафиксированное разбиение Titanic и готовый корректный конвейер `Pipeline`;
- сравните логистическую регрессию с неглубоким деревом с помощью стратифицированной кросс-валидации;
- вычислите precision, recall и F1 из матрицы ошибок;
- получите вневыборочные (out-of-fold, OOF) вероятности всех трёх кандидатов на одинаковых частях выборки и покажете таблицей, что правило выполнимо только для зафиксированного случайного леса без дополнительной настройки;
- выберете порог по правилу «recall ≥ 0,85; доля флагов ≤ 55%; затем максимум precision»;
- один раз оцените модель на внешней отложенной тестовой выборке и изучите ошибки по полу и классу;
- объясните, почему историческая ассоциация не превращается в причинный вывод или инструкцию спасателям.

## Подготовка

Нужен Python 3.12 или 3.13.

```bash
python3 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
python -m pip install -r requirements.txt
jupyter lab case-03.ipynb
```

Дело полностью автономно: корректный конвейер `Pipeline` и разбиение определены внутри тетради, а собственная копия `data/titanic.csv` входит в ZIP. Проходить II-02 технически не требуется, хотя его выводы служат контекстом.
