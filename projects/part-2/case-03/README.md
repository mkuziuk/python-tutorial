# Дело II-03. Цена одной ошибки

После удаления утечки у Бюро остаётся честная модель, но поставщик продолжает показывать только accuracy. В этом деле вы определите цену ложных решений, выберете порог по операционному правилу и проверите различия между группами.

## Результат дела

За 4–5 часов вы:

- независимо восстановите зафиксированный Titanic split и готовый честный pipeline;
- сравните логистическую регрессию с неглубоким деревом по stratified CV;
- вычислите precision, recall и F1 из матрицы ошибок;
- получите out-of-fold вероятности всех трёх кандидатов на одинаковых fold и докажете таблицей, что правило выполнимо только для фиксированного random forest без ученической настройки;
- выберете порог по правилу «recall ≥ 0,85; доля флагов ≤ 55%; затем максимум precision»;
- один раз откроете внешний holdout и изучите ошибки по полу и классу;
- объясните, почему историческая ассоциация не превращается в причинный вывод или инструкцию спасателям.

## Подготовка

Нужен Python 3.12 или 3.13.

```bash
python3 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
python -m pip install -r requirements.txt
jupyter lab case-03.ipynb
```

Дело полностью автономно: честный pipeline и split определены внутри тетради, а собственная копия `data/titanic.csv` входит в ZIP. Проходить II-02 технически не требуется, хотя его выводы служат контекстом.
