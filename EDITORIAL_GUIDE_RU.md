# Редакционный стандарт русскоязычного курса

Этот документ задаёт единый язык для страниц сайта, тетрадей Jupyter, README,
учебных комментариев и сообщений программ. Его задача — помочь писать по-русски,
а не переводить английский черновик слово за словом.

## Аудитория и тон

- Курс рассчитан на начинающих и продолжающих изучать Python и классическое
  машинное обучение.
- Обращайтесь к читателю на «вы». В инструкциях используйте прямые формы:
  «добавьте», «сравните», «запустите».
- «Мы» уместно, когда автор вместе с читателем разбирает идею, но не должно
  подменять точную инструкцию.
- Сюжетный язык оставляйте в завязке, переходах и финале. Технические разделы
  пишите спокойно и прямо, без рекламных преувеличений и канцелярита.
- Не меняйте даты, числа, критерии заданий, степень надёжности улик, границы
  выводов и другие факты, зафиксированные в `PLOT.md` и `PLOT_PART_2.md`.

## Русский текст и типографика

- Последовательно используйте «ё», кавычки-«ёлочки» и длинное тире.
- Не соединяйте русские слова дефисом с английским термином, если существует
  естественная русская формулировка: не `learner-набор`, а «учебный набор».
- Избегайте кальки, цепочек существительных и англоязычных слов в роли обычных
  членов русского предложения.
- Один объект называйте одинаково в главе, README, тетради и разборе решения.
- Сохраняйте оригинальное написание имён API, классов, функций, файлов, полей,
  команд и элементов интерфейса: `Pipeline`, `GridSearchCV`, `source_id`,
  `Restart Kernel → Run All`.
- Английский термин допустим при первом знакомстве, если он нужен для чтения
  документации: «отложенная тестовая выборка (holdout)». Дальше используйте
  русское название.

## Словарь курса

| Не использовать как основную формулировку | Основная формулировка |
| --- | --- |
| learner notebook | тетрадь для самостоятельной работы |
| notebook | тетрадь Jupyter; далее — тетрадь |
| train / test | обучающая / тестовая выборка |
| split | разбиение |
| holdout | отложенная тестовая выборка |
| baseline | базовая модель; точка отсчёта |
| pipeline | конвейер `Pipeline`, если речь об объекте sklearn; иначе процедура обработки |
| fold | часть выборки; после пояснения допустимо «фолд» |
| score | значение метрики |
| claim | заявление |
| target | целевая переменная |
| feature | признак |
| batch | партия данных |
| vendor | поставщик |
| deployment | внедрение |
| model card | карточка модели |
| bootstrap | бутстреп; при первом употреблении кратко пояснить пересэмплирование |
| grid search | перебор заданной сетки параметров; имя `GridSearchCV` не переводить |

Названия столбцов, ключей, переменных и значений из кода не переводятся. Например,
в тексте пишем «целевая переменная», но поле `target` оставляем без изменений.

## Учебные комментарии в коде

- Комментарий объясняет причину решения, границу данных, единицы измерения,
  предположение, инвариант или неочевидное поведение библиотеки.
- Не пересказывайте очевидную операцию строка за строкой и не комментируйте
  импорты, простые присваивания и служебный код без учебной ценности.
- Перед логическим блоком предпочтителен один короткий комментарий вместо серии
  пояснений справа от каждой строки.
- В материалах для самостоятельной работы комментарий задаёт ориентир, но не
  раскрывает вызов API, формулу или последовательность действий из ответа.
- В тетрадях комментарий вне `# BEGIN SOLUTION` / `# END SOLUTION` должен быть
  безопасен для учебной версии. Объяснение, раскрывающее решение, помещайте
  внутрь маркеров.
- Не добавляйте обязательные docstring ко всем функциям: комментарий нужен там,
  где без него легко нарушить смысл процедуры.

## Границы редакторской правки

Можно переписывать Markdown, учебные комментарии и пользовательские сообщения,
если сохраняются их смысл и структура. Нельзя менять идентификаторы, API, пути,
ссылки, схемы, вычисления, пороги, размеры выборок, ожидаемые значения, задания,
данные и выводы расследований. Если естественная фраза требует догадки о смысле,
оставьте исходный текст и отметьте место для автора.

## Самопроверка редактора

1. Фраза звучит так, будто её сразу написали по-русски.
2. Термин соответствует словарю и одинаково назван во всех копиях материала.
3. Технический смысл, числа и границы вывода не изменились.
4. Комментарий помогает понять решение, а не повторяет синтаксис.
5. Учебная версия не получила подсказку, которая решает упражнение за читателя.
